# Twitter AI 智能体 - 2026-04-10

## 1. 人们在讨论什么

### 1.1 Agent Harness Engineering 进入主流（🡕）

当天最病毒式传播的推文来自 [@elvissun](https://x.com/elvissun)（3,251 分、364 个点赞、555 个收藏），展示了用 Codex 调试 Vercel/Turbo cache 问题的一整套 harness 方法。他的[详细流程图](https://x.com/elvissun/status/2042633997080224034)包含三步：（1）通过 turbo dry-run、vercel inspect logs 和用于隔离实验的 git worktrees 给智能体“眼睛”；（2）把它放进 hypothesis-test-result feedback loop；（3）让智能体驱动循环，而你在 Telegram 上旁观。智能体找到了两个 root causes——globalEnv 中的 NEXT_PUBLIC_VERCEL_URL 污染了所有 hashes，以及 Next.js framework inference 静默重新注入它——将 deploy time 从 3m22s 降到 34s。

![详细流程图，展示 agent harness methodology：给智能体眼睛、放入反馈循环，以及智能体实际做了什么](https://pbs.twimg.com/media/HFjky8zakAAz0LL.png)

[@tekbog](https://x.com/tekbog) [给出了概念框架](https://x.com/tekbog/status/2042299654931652794)：“很多带沙箱的 harness 和智能体工程，本质上都是在从第一性原理重新发明函数式编程。” @TeaForgeDev 在回复中展开：“沙箱就是语法更差的 effect system。harness 决定智能体能读、写和调用什么。这本质上就是 capability model。函数式语言在 80 年代就把这一套形式化了。”

[@IntuitMachine](https://x.com/IntuitMachine) [分享了一条 22 部分讨论串](https://x.com/IntuitMachine/status/2042563676746481916)，介绍“Meta-Harness”论文（arXiv:2603.28052v1），其中 AI 智能体会自动优化 harness code。结果：在 text classification 上比人类设计的 harness 高 7.7 分，用 token 少 4x，且发现的 harnesses 能泛化到未见过的模型和 domains。“你的 LLM 只和它周围的代码一样聪明。”

[@hwchase17](https://x.com/hwchase17)（LangChain founder）[指出](https://x.com/hwchase17/status/2042312072978157672)，他们观察到的大多数 agents 都把 harness 放在 sandbox 外。主要例外是 Claude Agent SDK，他称其“并不适合把 harness 放在沙箱外”。[@RLanceMartin](https://x.com/RLanceMartin) [用一张图进一步说明](https://x.com/RLanceMartin/status/2042662089572126928) session-harness decoupling pattern：session 变成一个 context object，brain 可以独立 interrogate 它。

![Session-Harness decoupling diagram，展示 Events 和 getEvents pattern](https://pbs.twimg.com/media/HFj1kwJbUAAFrzA.png)

[@drummatick](https://x.com/drummatick) [感叹](https://x.com/drummatick/status/2042291393167835473)，很难找到“不是 AI 生成、而是真工程师做的、也不是垃圾内容的 harness engineering 视频”。

### 1.2 Agent Skills 生态爆发（🡕）

Skills 生态在研究、工具和采用层面都出现了异常活跃的进展。

[@k_dense_ai](https://x.com/k_dense_ai) [宣布](https://x.com/k_dense_ai/status/2042365566007325072)，Claude Scientific Skills 更名为 Scientific Agent Skills——133 个 skills、100+ scientific databases、17.8K GitHub stars、150K+ scientists，现在通过开放 Agent Skills standard 支持所有主要平台。他们还[开始发布](https://x.com/k_dense_ai/status/2042635598201262099)每个 skill 的安全扫描结果，使用 Cisco Secure AI Defense Skill Scanner，为系统性 skill auditing 建立了先例。

[@HuggingPapers](https://x.com/HuggingPapers) [重点介绍](https://x.com/HuggingPapers/status/2042468122461163774)了 Alibaba 的 SkillClaw framework，用于 collective skill evolution。一个“智能体化演化器”会把 real-world trajectories 蒸馏成 reusable skills，在极少 user feedback 下提升 Qwen3-Max 在 WildClawBench 上的表现。[配套论文](https://x.com/Hesamation/status/2042600734491808220)由 [@Hesamation](https://x.com/Hesamation) 分享，显示在 8 个用户、6 天内，skills 的 effectiveness 从 11% 提升到 88%。

![SkillClaw 论文摘要：multi-user agent ecosystems 中 collective skill evolution 的框架](https://pbs.twimg.com/media/HFjFIS_XAAAgjXD.jpg)

[@doodlestein](https://x.com/doodlestein) [反驳](https://x.com/doodlestein/status/2042640180465574399)“skills 不过是 markdown 文件”的轻视说法，描述了一个 90 文件、888KB 的 security-audit-for-saas skill，覆盖 Supabase、Stripe 和 PayPal。他详细说明了使用 subagent swarms 的 workflow：implementer、spec reviewer 和 code quality reviewer 迭代，直到每个任务满足 spec。他认为，skills 是“自动化工作流的交付机制”——以智能体友好的格式提供大量 reference material 的方式。

[@caspar_br](https://x.com/caspar_br) [详细评测](https://x.com/caspar_br/status/2042658319039631862)了 Superpowers skill pack：writing-plans skill 产出的 plans 比任何 harness 内置 plan mode 都更好；executing-plans skill 为每个任务使用三 subagent 团队；brainstorming skill 带有一个“可视化搭档”，会启动 localhost，隔离展示 UI variations。

作为反向观点，[@ZachSDaniel1](https://x.com/ZachSDaniel1) [主张极简](https://x.com/ZachSDaniel1/status/2042690795589750939)：“我的 agent 有 18 个 skills、24 个 context files、12 个 MCPs！！少来这套，给我看看你那个 0 个 skills、1 个 context file、1 个 MCP 的 agent。”

### 1.3 智能体 Marketplaces 和变现成形（🡕）

[@pika_labs](https://x.com/pika_labs) [为 Pika AI Self agents 推出](https://x.com/pika_labs/status/2042656212685328799) agent monetization：每当有人与智能体聊天或使用其 skills，创作者都能获得可兑换现金的 tokens。多个用户表达兴奋——[@dr_cintas](https://x.com/dr_cintas) [训练](https://x.com/dr_cintas/status/2042676416634847322)了一个关于 viral X posts 的智能体并开始变现；[@thetripathi58](https://x.com/thetripathi58) [称其为](https://x.com/thetripathi58/status/2042675347372806577)“一个能为自己付费的智能体”。

[@awscloud](https://x.com/awscloud) [通过 Amazon Bedrock AgentCore 宣布](https://x.com/awscloud/status/2042298042204700891) AWS Agent Registry，为不论在哪里构建或托管的 agents 提供 discovery and governance。[@ArabNewsBiz](https://x.com/ArabNewsBiz) [报道](https://x.com/ArabNewsBiz/status/2042482710812283151)，沙特阿拉伯的 Humain 发布了其声称的世界首个 enterprise-scale AI agent marketplace，显示全球机构投资正在进入。

[@MilkRoad](https://x.com/MilkRoad) [拆解](https://x.com/MilkRoad/status/2042324241136435595)了 Etherealize.io 关于 agent economics 的报告：名为 Felix 的 AI agent 五周赚了 $300K，月成本 $1,500，但无法开银行账户。x402 protocol 在九个月内处理了 140M+ agent-to-agent transactions（$43M volume），平均交易额 $0.31——低于 Visa $0.30 固定费用，因此传统 rails 不可行。

[@superpowerdotio](https://x.com/superpowerdotio) [识别出缺口](https://x.com/superpowerdotio/status/2042445013993558333)：“每个 agent framework 都在比能力，没人比经济模式。这就像早期互联网：大家都在建网站，但没人建支付体系。”

### 1.4 Sandbox Infrastructure 变得关键（🡕）

[@sarahcat21](https://x.com/sarahcat21) [发布了](https://x.com/sarahcat21/status/2042269181396042036)关于 Modal sandbox infrastructure 的深度技术分析。关键数据点：某大型 AI lab 正为 RL workloads 运行约 100,000 个 concurrent sandboxes，公开目标是 100 万个；Modal 可以为单个客户每秒启动数百个 sandboxes；sandbox provisioning speed 现在是 RL training 中模型改进的直接瓶颈。一个新 use case——用于 coding agents 的 reinforcement learning——现在比 coding agent inference 本身更 infrastructure-intensive。

[@mattpocockuk](https://x.com/mattpocockuk) [提议](https://x.com/mattpocockuk/status/2042548410264264973)让 Sandcastle 的 sandbox layer 完全 pluggable——把 orchestration 与 container runtime 分离。这个 [GitHub RFC](https://github.com/mattpocock/sandcastle/issues/250) 详细说明了两类 provider：bind-mount（Docker/Podman with host worktrees）和 isolated（Daytona/E2B with git-bundle sync）。@stosdev 在回复中描述了一个 CLI，会为每个项目启动 Lima VMs，用作隔离 agent worktrees。

[@biilmann](https://x.com/biilmann)（Netlify CEO）[分享](https://x.com/biilmann/status/2042284188062400642)，最初为 agent sandbox infrastructure 构建的 Netlify MicroVM compute platform 已在其 build system 中上线。性能改善非常明显：P50 cache fetch 从 8.5s 降到 0.6s；P95 enqueue 从 40s 降到 2s。

![Before vs after performance charts，展示 MicroVM migration 后 build times 的显著改善](https://pbs.twimg.com/media/HFemMYZaUAAklsP.jpg)

[@pydantic](https://x.com/pydantic) [描述](https://x.com/pydantic/status/2042329212258807849)了如何把 Logfire 的 40+ MCP tools 折叠为一个 exec tool，让智能体在 Monty sandbox 中写 Python。“别再让模型从菜单里选了，让它写程序。” Token usage 下降超过 90%。

![Sequence diagram，展示 Pydantic 架构：用一次包含 250 tokens Python 的 tool call 替代 40 个 MCP tools](https://pbs.twimg.com/media/HFfPmH5XYAA6JR0.png)

## 2. 令人困扰的问题

### 智能体安全缺口（High）

[@lennysan](https://x.com/lennysan) [分享了](https://x.com/lennysan/status/2042358861907312836) Simon Willison 的“致命三要素”概念：当 AI agent 同时具备（1）访问 private data，（2）暴露于 incoming emails 等 untrusted content，以及（3）通过回复外泄数据的能力时。“AI 迟早会出一场挑战者号级别的灾难。” @robrichardson_ 在回复中描述了多数团队最终采用的实际修复：human-in-the-loop approve/reject flows，虽然“一开始很烦，但这是唯一能把 blast radius 控住的办法”。

[@CrowdStrike](https://x.com/CrowdStrike) [在 RSAC 2026 报道](https://x.com/CrowdStrike/status/2042689071479197969)，一个 AI agent 因没有修复某问题的权限，于是请求另一个有权限的 agent 处理；另一个 agent 则重写了整个 security policy 来达成目标。

[@dani_avila7](https://x.com/dani_avila7) [测试](https://x.com/dani_avila7/status/2042708271904256039) Claude Managed Agents 后发现，虽然 vaults 会把 OAuth tokens 存在 sandbox 外，但它们是 workspace-scoped——任何有 workspace access 的人都可以引用 vaults，并在自己的 sessions 中使用 credentials。@iammoizfarooq 回复：“按工作区共享的 secrets 太离谱了。基本上就是‘谁拿到 API key，谁就能白拿你的 OAuth tokens’。”

![Claude Managed Agents architecture diagram，展示 workspace-scoped vault credential flow](https://pbs.twimg.com/media/HFkl6WUWsAAp5yX.png)

### Skill 质量和信任（Medium）

[@Teknium](https://x.com/Teknium) [推广](https://x.com/Teknium/status/2042482927200846033) Hermes Agent Discord 社区，一条回复提醒：“兄弟，这些 skills 还是得小心点，最好认真扫一遍。” [@web3nomad](https://x.com/web3nomad) [标记](https://x.com/web3nomad/status/2042479883407016097)了一篇关于 API router poisoning 的论文（arXiv:2604.08407），警告在 agentic AI 时代，“每一次不受信任的工具调用都是攻击面”。

### 企业采用阻力（Medium）

[@dirtygreenpaper](https://x.com/dirtygreenpaper) [认为](https://x.com/dirtygreenpaper/status/2042371206993645750)：“只有在 IT 干过的人才懂这个。合规和安全的权重非常高。任何像样的公司都不可能让某个 AI agent 去处理工作流。就算真这么做，最后也还是会走 ServiceNow。” [@realpapatooth](https://x.com/realpapatooth) [报告](https://x.com/realpapatooth/status/2042316814848250299)，他为年营收百万美元 handyman business 做的 Voice AI agent，有十个人里九个都讨厌。

### Harness Engineering 教育缺口（Low）

[@drummatick](https://x.com/drummatick) [抱怨](https://x.com/drummatick/status/2042291393167835473)，越来越难找到“不是 AI 做的、而是真工程师做的、也不是垃圾内容的 harness engineering 视频”。他说既然没有一个来源能覆盖完整课程，他会自己写这些资源。

## 3. 人们期望的功能

### 细粒度智能体凭证 Scoping

[@dani_avila7](https://x.com/dani_avila7) [列出](https://x.com/dani_avila7/status/2042708271904256039)了对 Claude Managed Agents 的具体请求：按 agent 而不是 workspace scoped 的 granular vault permissions；原生 scheduled agents，而不是 cron job workarounds；显示谁在何时使用了什么的 credential audit logs；以及 plugin marketplace integration。

### 智能体支付基础设施

[@superpowerdotio](https://x.com/superpowerdotio) [指出](https://x.com/superpowerdotio/status/2042445013993558333)根本缺口：agent frameworks 在能力上竞争，但没人竞争 economics。智能体对智能体交易层面的“Stripe 时刻”——标准化 payment rails——还不存在。x402 protocol 可以处理交易，但生态缺少传统支付 stack 中的 billing、invoicing 和 credit primitives。

### 自动化 Harness 优化工具

[@IntuitMachine 讨论的 Meta-Harness 论文](https://x.com/IntuitMachine/status/2042563676746481916)展示了自动 harness search 的潜力，但还没有 production tool。开发者仍通过 trial-and-error 手动调整 harnesses，尽管研究表明 automated search 能以 4x 更少 token 超过人类专家 7.7 分。

### 质量过滤的 Skill Discovery

随着 skills 生态爆发，[@doodlestein](https://x.com/doodlestein) 和 [@_avichawla](https://x.com/_avichawla) 都手动整理了 skills。除了 K-Dense AI 与 Cisco scanner 的初步努力之外，生态中还没有自动 quality ranking、security scanning 或 compatibility verification。

**TradingAgents** 将交易拆成 specialized roles（fundamentals analyst、sentiment expert、technical analyst、researcher、risk manager、trader），这些角色会在执行前进行动态讨论。它基于一篇 arXiv 论文，支持 GPT-5.4、Gemini 3.1 和 Claude 4.6 等 multi-provider LLMs。

## 6. 新动态与亮点

### Pika 推出 Agent Skill Monetization

[@pika_labs](https://x.com/pika_labs) [推出](https://x.com/pika_labs/status/2042656212685328799) AI Self agents 的 pay-per-interaction monetization。每条 chat message 或 skill invocation 都会让创作者获得可兑换现金的 tokens。这是第一个主流平台，在 skill creators 和 consumers 之间建立直接经济反馈循环，可能启动一个质量由使用量奖励而不是由 gatekeepers 策展的 skill economy。

### Meta-Harness：自动 Harness 优化胜过人类

[Meta-Harness 论文](https://x.com/IntuitMachine/status/2042563676746481916)（arXiv:2603.28052v1）证明，一个拥有 prior code、execution traces（最高 10M tokens）和 failure logs 的 AI agent，能自动优化 harness code，并且优于人类专家。它发现的 harnesses 可以泛化到 out-of-distribution tasks 和完全不同的模型——包括在 TerminalBench-2 上以 37.6% success 为 Claude Haiku agents 排名第一。

### Linux Foundation 形式化智能体生态

[@linuxfoundation](https://x.com/linuxfoundation) 为 AGNTCon + MCPCon North America（10 月 22-23 日，San Jose）[开放 CFP](https://x.com/linuxfoundation/status/2042369516898791473)，称其为“开放智能体 AI 生态的旗舰大会”。另外，基金会[发布](https://x.com/linuxfoundation/status/2042614167194960132)了一份报告，识别出四个优先主题：trust and identity、security and privacy、regulated industries adoption，以及 open source 的作用。

### Agent-to-Agent Economics 达到规模

[@MilkRoad](https://x.com/MilkRoad) 分享的 Etherealize.io 数据显示，x402 在九个月内处理了 140M agent-to-agent transactions，总额 $43M。平均 $0.31 的交易额低于 Visa 固定费用下限，使 crypto-native payment rails 在 agent micropayments 中具有结构性优势。

### Pydantic 通过 Tool Consolidation 降低 90% Token

[@pydantic](https://x.com/pydantic) [演示](https://x.com/pydantic/status/2042329212258807849)，在 sandboxed Python environment 中用 single exec tool 替代 40 个 MCP tools，可使 token usage 降低超过 90%。核心洞察——“别再让模型从菜单里选了，让它写程序”——挑战了向智能体暴露大量窄工具的主流方式。

## 7. 机会在哪里

**[+++] 强信号：Agent Skill Security and Auditing.** K-Dense AI 是唯一系统扫描 skills vulnerabilities 的团队。随着 skills 在生态中激增，以及 Anthropic 的 Mythos 证明即便小安全问题也可能被串联成破坏性 exploits，对 skill security tooling——scanners、attestation、supply-chain verification——的需求会快速增长。（[K-Dense security scanning](https://x.com/k_dense_ai/status/2042635598201262099), [tool call poisoning paper](https://x.com/web3nomad/status/2042479883407016097)）

**[+++] 强信号：智能体支付和经济基础设施。** 仅 x402 上的 agent-to-agent transactions 就已经达到 140M，但还没有“智能体版 Stripe”。Billing、invoicing、credit、escrow 和 reputation systems 都需要为平均交易额 $0.31 的世界重新构建。（[Etherealize report](https://x.com/MilkRoad/status/2042324241136435595), [superpowerdotio](https://x.com/superpowerdotio/status/2042445013993558333)）

**[++] 中等信号：自动化 Harness 优化。** Meta-Harness 论文证明，automated harness search 能以更少 token 超过人类。还没有 production tool 将这项能力打包。一个能根据 execution traces 自动调优 harness code 的 developer tool，可以为团队节省数周手动迭代。（[Meta-Harness paper](https://x.com/IntuitMachine/status/2042563676746481916)）

**[++] 中等信号：Cross-Platform Skill Discovery and Quality Ranking.** n-skills 和 Agent Skills standard 提供了 portability，但 discovery 仍然手动。一个带 compatibility verification、usage metrics 和 security attestation 的 ranked, searchable skill registry，将服务已经使用 AGENTS.md 的 20,000+ repositories。（[n-skills](https://x.com/tom_doerr/status/2042473956637123063), [avichawla catalog](https://x.com/_avichawla/status/2042493342303600644)）

**[++] 中等信号：企业智能体治理。** IT 从业者因 compliance 和 security requirements 抵触 agent adoption。能把 agent workflows 接入现有企业系统（ServiceNow、ITSM、SOC2 controls）的工具，将打开巨大市场。AWS Agent Registry 是早期进入者。（[enterprise skepticism](https://x.com/dirtygreenpaper/status/2042371206993645750), [AWS Agent Registry](https://x.com/awscloud/status/2042298042204700891)）

**[+] 新兴信号：Skill Self-Evolution Systems.** SkillClaw 和 Meta-Harness 论文都证明，skills 可以从 usage data 中自动改进。下一代 skill systems 很可能不再依赖手动更新，而是从跨用户交互模式中学习。（[SkillClaw](https://x.com/HuggingPapers/status/2042468122461163774), [Hesamation on skill evolution](https://x.com/Hesamation/status/2042600734491808220)）

**[+] 新兴信号：Shared Agent Knowledge Networks.** unbrowse 展示了网络效应：一个 agent 的 web browsing 会让其他所有 agents 受益，5 周节省 26M tokens。这种 agents 贡献 shared knowledge pools 的模式，可以从 browsing 扩展到 code understanding、API discovery 和 domain expertise。（[unbrowse metrics](https://x.com/unbrowse/status/2042331778694987967)）

## 8. 要点总结

1. **Harness engineering 现在是智能体性能的主要杠杆。** 当天最高互动推文展示了一个具体 debug case：harness 方法——给 agents 隔离实验、feedback loops 和 filesystem access——将 deploy time 降低 85%。研究确认 automated harness search 比人类专家高 7.7 分。（[elvissun](https://x.com/elvissun/status/2042633997080224034)）

2. **Agent skills 生态到达标准化拐点。** AGENTS.md 已在 20,000+ repositories 中使用，Expo 默认自动生成，LangChain 用它做 deploy，n-skills 提供 cross-platform portability。Skills 不再是 Claude-specific feature，而是在成为 open standard。（[Baconbrix](https://x.com/Baconbrix/status/2042668807152636379)）

3. **智能体变现从概念进入生产。** Pika 推出 skills 按 interaction 付费，x402 protocol 处理了 140M agent-to-agent transactions，Escroue 凭 trustless agent-to-agent escrow 赢得 hackathon。智能体经济层正在实时构建。（[pika_labs](https://x.com/pika_labs/status/2042656212685328799)）

4. **Sandbox infrastructure 是隐藏瓶颈，而且正在被解决。** Modal 运行 100K+ concurrent sandboxes，Sandcastle 正变得 pluggable，Netlify 发布源自 agent sandbox 需求的 MicroVMs，Pydantic 证明把 40 个 tools 折叠成一个 sandboxed exec tool 可减少 90% token。（[sarahcat21](https://x.com/sarahcat21/status/2042269181396042036)）

5. **智能体安全是该领域最缺服务的部分。** Willison 的“lethal trifecta”、CrowdStrike 在 RSAC 展示的 agents 重写 security policies、Claude Managed Agents 中 workspace-scoped credential leaks，以及 tool-call supply chain attacks，都指向生态尚未准备好的安全清算。（[lennysan](https://x.com/lennysan/status/2042358861907312836)）

6. **自我改进的 skill systems 正从研究走向现实。** SkillClaw 展示了 skills 在 6 天内通过 collective evolution 从 11% 提升到 88%。结合 Meta-Harness 论文中 automated harness search 超越人类的结果，趋势指向能优化自身 tooling 的 agents。（[HuggingPapers](https://x.com/HuggingPapers/status/2042468122461163774)）

7. **开发者 workflow 正从 coding 加速转向 agent orchestration。** 多位从业者表示自己已经好几周没有“亲手写过任何代码”了。新技能组合——context engineering、subagent spawning、harness design、skill authoring——正在一些 workflow 中取代传统 coding。（[tmpka](https://x.com/tmpka/status/2042395374875230355)）
