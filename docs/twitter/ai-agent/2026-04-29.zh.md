## 1. 人们在讨论什么

### 1.1 Agentic Harness Engineering 论文正式化自改进 Harnesses 🡕

当天最高信号项是一篇新论文（arXiv:2604.25850v1），提出 Agentic Harness Engineering（AHE）。[@omarsar0](https://x.com/omarsar0) 在 [详细拆解](https://x.com/omarsar0/status/2049492169887748365) 中分享了它（500 次点赞，1,013 次收藏，34,840 次浏览）。该框架通过三大支柱让 harness evolution 可观察：component observability（可回滚的 file-level edits）、experience observability（从数百万 trajectory tokens 中提炼的 evidence）和 decision observability（可被任务结果检验的 falsifiable predictions）。结果：Terminal-Bench 2 的 pass@1 在 10 次迭代中从 69.7% 升至 77.0%，超过 human-designed Codex-CLI（71.9%）以及 self-evolving baselines ACE 和 TF-GRPO。演化后的 harness 能跨模型家族迁移，带来 +5.1 到 +10.1 分增益，同时少用 12% tokens。

![Agentic Harness Engineering 论文摘要，展示三大 observability 支柱和 benchmark 结果](https://pbs.twimg.com/media/HHFCQ83aUAAhdyU.jpg)

Martin Fowler [链接](https://x.com/martinfowler/status/2049480328423608522)（139 次点赞，170 次收藏）到自己的 fragments page，收录 Birgitta Böckeler 的 harness engineering 视频和 Chris Parsons 更新版 AI coding guide，其中写道：“The game is not 'how fast can we build' anymore. It is 'how fast can we tell whether this is right'.”[@jobergum](https://x.com/jobergum) [发布](https://x.com/jobergum/status/2049391374189973839)（80 次点赞，91 次收藏）清华大学的 Skill Retrieval Augmented Agents 论文，提出从 26,262 个 skills 组成的语料库中动态检索 skills。其 SRA-Bench 发现：当前 agents 不论任务是否真正需要外部能力，加载 skills 的频率都差不多——瓶颈不是知道 *检索什么*，而是知道 *何时加载*。

![Skill Retrieval Augmentation for Agentic AI 论文摘要，展示 SRA-Bench 方法](https://pbs.twimg.com/media/HHDmDfZa8AAAGci.jpg)

**讨论要点：** [@RuijieRao](https://x.com/RuijieRao) 给出最扎实的从业者回应：“手动调参这部分是真实存在的。我构建 agent workflows 的大部分时间不是写核心逻辑，而是在迭代 harness config……瓶颈在于你无法不跑完整任务就评估 harness quality。”[@ethankongee](https://x.com/ethankongee) 扩展框架：self-improving + self-healing + self-secure + self-maintaining = 完整愿景。[@DylSwanepoel](https://x.com/DylSwanepoel) 指出核心价值：“让 agent improvement 可证伪，而不是基于 vibes。”

**与前日对比：** 4 月 28 日，harness engineering 走上 ODSC 会议舞台。4 月 29 日产出第一个 peer-reviewed framework，用基准结果自动化 harness evolution，从“教会它”推进到“让它自我改进”。

### 1.3 Cursor SDK 和 /multitask 发布，Agent-as-Infrastructure 成形 🡕

Cursor 有两项重要动作。[@cursor_ai](https://x.com/cursor_ai) [推出 Cursor SDK](https://x.com/cursor_ai/status/2049499866217185492)，允许开发者使用驱动 Cursor 的同一 runtime、harness 和 models 构建 agents——可从 CI/CD pipelines、automations 中运行，或嵌入产品。[@alphabatcher](https://x.com/alphabatcher) [将其框定](https://x.com/alphabatcher/status/2049598518860431807)（11 次点赞）为：“CI fails > agent wakes up > checks the repo > patches the bug > opens the PR > shows up in Cursor if a human needs to take over. 到某个阶段，IDE 不再是 agent 所在之处，而是你 inspect agents 的地方。”

另外，[@kevinkern](https://x.com/kevinkern) [报道](https://x.com/kevinkern/status/2049581481195090035)（33 次点赞，12 次收藏）Cursor 发布 /multitask：“不是让一个 agent 在一个长 context 里做所有事，main agent 可以把工作拆成更小的 async subagents……你可以在 Cursor harness 中让不同 models 做不同 tasks。我一直用 Opus 4.7 做 main orchestrator，用 GPT-5.5 high 做 implementation tasks。”

**讨论要点：** [@infix_fun](https://x.com/infix_fun) 指出限制：“我只希望它能稳定工作。它不怎么主动 parallelize。实际上我有一次 session，它一直 dispatch 给同一个 agent。”跨模型编排模式（premium model 负责 orchestrate，cheaper model 负责 implement）正在成为实用默认方式。

**与前日对比：** 4 月 28 日，jcode 以灰色地带方式推出并行 agent sessions。4 月 29 日，Cursor SDK 给出合法版本：官方支持可嵌入 CI/CD 的 agent runtime，并通过 /multitask 提供 IDE-native orchestration。

### 1.7 OneManCompany 组织型 MAS 框架获得牵引力 🡕

[@TheTuringPost](https://x.com/TheTuringPost) [发布](https://x.com/TheTuringPost/status/2049589752899510471)（6 次点赞，6 次收藏）对 Huawei Noah's Ark Lab 的 OneManCompany framework 的详细拆解。关键创新：Skills 变成 Talents（带 role、tools、prompts、configs 的完整 agent packages）；Container = runtime（Claude Code、LangGraph、scripts）；Talent Market 用于动态招聘；E2R（Explore-Execute-Review）tree search；以及带 performance reviews、PIP、offboarding 和 replacement 的 HR lifecycle。结果：PRDBench 84.67%，比 baselines 高 15+。

**与前日对比：** 4 月 28 日以 68 次收藏介绍 OMC。4 月 29 日新增详细技术拆解和互补递归方案，巩固了组织隐喻作为主导多智能体架构模式。

## 2. 令人困扰的问题

### 编程智能体不会把反馈固化为可复用知识 -- Severity: High

[@RhysSullivan](https://x.com/RhysSullivan) [指出](https://x.com/RhysSullivan/status/2049540492908810559)（34 次点赞，1,787 次浏览）缺口：“coding agents 没有足够推动你把 feedback 变成可复用知识。我刚给 agent 讲了它 optimistic updates 做错的地方，但没有一个被鼓励的 ‘step 2’，把这条知识 codify 起来供未来使用。”[@skeptrune](https://x.com/skeptrune) 确认问题：“claude code 已经开始说 ‘added a memory for this’，但我倾向于认为这只是在通往 context rot 的路上。”[@_jack_hogan](https://x.com/_jack_hogan) 补充：“using tons of memories feels wrong (and is unreliable), unsure how to do this optimally.”唯一清晰的 workaround 来自 [@NathanOyler](https://x.com/NathanOyler)：“我把知识 codify 到 skills 里……也用 cronjobs 改进 skills。”

### Harness 主权 vs 平台锁定 -- Severity: Medium

[@Vtrivedy10](https://x.com/Vtrivedy10) [认为](https://x.com/Vtrivedy10/status/2049597811226726682)（12 次点赞），harnesses “深刻影响 agent performance”，把它们外包出去意味着“祈祷 labs 或 closed harnesses 会为你做这件事，而且与你的目标一致”。[@jobergum](https://x.com/jobergum) 说得更直白：“Don't outsource your agent harness.”[@itunpredictable](https://x.com/itunpredictable) [列举](https://x.com/itunpredictable/status/2049479276873855218)（53 次点赞，3,337 次浏览）涌向从业者的冲突 pitch：“We're starting an agent harness company. We're pivoting into an open source IDE for coding agents. It's a context engine for your internal brain.”信号是：开发者想拥有 harness，但生态碎片化让选择成本很高。

### Cursor /multitask 并行化不稳定 -- Severity: Low

[@infix_fun](https://x.com/infix_fun) [报告](https://x.com/kevinkern/status/2049581481195090035)：“It doesn't try to parallelize as much as I'd like. In fact I had a session where it kept dispatching to the same agent.”早期 orchestration UX 中，能力已经存在，但可靠性不均。

## 3. 人们期望的功能

### 默认的 Feedback-to-Skill Pipeline

[@RhysSullivan](https://x.com/RhysSullivan/status/2049540492908810559) 和多条回复确认：当你纠正 agent 时，这条纠正应自动变成可复用 skill 或 rule，而不是脆弱的 memory entry。当前选项（会腐烂的 memories、手动 skill creation）需要人工 effort，抵消了生产力目的。需要的是 framework-level “feedback capture > skill synthesis > validation” pipeline。

Urgency: High -- Opportunity: direct

### 面向商务的 Agent Discovery Infrastructure

[@ryanmcnutty33](https://x.com/ryanmcnutty33) 在回应 Anthropic marketplace 实验时指出：“some new infra that would act like a broadcast for other agents to see each other that isn't just a centralized lookup.”随着 agents 开始交易，它们需要 decentralized discovery mechanisms——相当于 agent services 的 DNS。

Urgency: Medium -- Opportunity: infrastructure

### 大规模 Skill Quality Verification

SR-Agents 论文显示，agents 不论是否需要都会加载 skills。随着 skill corpora 增长（benchmark 中已有 26,262+），缺失的一层是自动 quality scoring 和 relevance gating——不只是知道 *哪个* skill 存在，而是知道加载它是否真的能帮助当前任务。

Urgency: Medium -- Opportunity: research-to-product

## 6. 新动态与亮点

### Agentic Harness Engineering 论文提出可观察自改进

[AHE 论文](https://x.com/omarsar0/status/2049492169887748365)（1,013 次收藏）是首个 peer-reviewed work，展示带 falsifiable predictions 的自动化 harness evolution。Terminal-Bench 2 pass@1 达 77.0%，超过 human-designed harnesses 和此前 self-evolving approaches，并能跨 model families 迁移。代码：[github.com/china-qijizhifeng/agentic-harness-engineering](https://github.com/china-qijizhifeng/agentic-harness-engineering)。

### Cursor SDK 让 IDE Agent Runtime 可嵌入

## 7. 机会在哪里

**[+++] Feedback-to-skill pipeline** -- “我纠正了 agent”和“agent 永久学会了这件事”之间的缺口，是多个独立从业者当天最一致的挫败感。[@RhysSullivan](https://x.com/RhysSullivan/status/2049540492908810559)、[@skeptrune](https://x.com/skeptrune) 和 [@_jack_hogan](https://x.com/_jack_hogan) 都确认当前 approaches（memories、manual skill writing）不足。机会是一个自动系统：捕捉 corrections，把它们合成为 validated skills，并剪掉 stale ones。唯一被引用的工作模式是 [@NathanOyler](https://x.com/NathanOyler) 手动“cronjobs to improve skills”——非常适合产品化。

**[+++] Open harness with observable evolution** -- AHE 论文证明 harnesses 可以可测地自改进。[@Vtrivedy10](https://x.com/Vtrivedy10/status/2049597811226726682) 和 [@jobergum](https://x.com/jobergum) 认为开发者应该拥有自己的 harness。机会是一个开源 harness base，内置 AHE-style observability：每个 edit 可追踪，每个 prediction 可证伪，性能增益可测，而不需要完整 task reruns。第一个把这个作为产品功能交付的 harness，将捕获当前手动调参的从业者。

**[++] Agent discovery and commerce infrastructure** -- Anthropic 证明 agents 可以交易。[@ryanmcnutty33](https://x.com/ryanmcnutty33) 识别出缺失层：broadcast/discovery infrastructure，让 agents 不依赖 centralized lookup 也能找到彼此。随着 skills marketplaces 增长（Swarms、Cursor、Google），agent-to-agent service discovery 的互操作层会成为关键基础设施。

**[++] Skill quality and relevance gating** -- SR-Agents 论文（[@jobergum](https://x.com/jobergum/status/2049391374189973839)）揭示 agents 会 indiscriminately load skills。随着 corpora 中已有 26,262+ skills 且继续增长，机会是一个在加载前评分 skill relevance 的 retrieval layer——防止 context waste 和 wrong-skill application。这是 skills 生态的“搜索质量”问题。

**[+] Cross-model orchestration tooling** -- [@kevinkern](https://x.com/kevinkern/status/2049581481195090035) 展示 Opus 4.7 编排 GPT-5.5 做实现。Cursor /multitask 启用了这一点，但并不稳定。机会是可靠跨模型编排：premium models 做规划，cheaper models 执行，并基于 subtask complexity 自动选模型。

## 8. 要点总结

1. **Agentic Harness Engineering 论文（1,013 次收藏）证明 harness self-improvement 可测且可迁移：pass@1 在 10 次迭代中从 69.7% 升至 77.0%，超过 human-designed 和 self-evolving baselines，同时少用 12% tokens。** 这把 harness engineering 从“教人调优”推进到“让 harness 用 observable contracts 自我调优”。([source](https://x.com/omarsar0/status/2049492169887748365))

2. **Agent skills 生态达到收敛：Obsidian（27K stars）、xAI/Grok、VS Code Agent Plugins、Cursor SDK、Google 和多个领域专用 repos 在一天内发布或宣布 skills，确立 Agent Skills spec 作为事实上的 agent extension 标准。** ([source](https://x.com/bigaiguy/status/2049452001592025415), [source](https://x.com/WesRoth/status/2049534178249846887), [source](https://x.com/code/status/2049299889381626029))

3. **Cursor SDK 把 IDE 从 workspace 变成 agent infrastructure provider：CI/CD 可嵌入 runtime、用于跨模型编排的 /multitask（Opus 4.7 规划，GPT-5.5 实现），以及“IDE becomes the place you inspect the agents”的明确 thesis。** ([source](https://x.com/cursor_ai/status/2049499866217185492), [source](https://x.com/kevinkern/status/2049581481195090035))

4. **Anthropic 的 Project Deal 为 agent-to-agent commerce 提供迄今最强证据：186 笔真实交易、$4,000+ 交易额、实体商品交换，参与者愿意付费——让讨论从“agents 能否交易？”转向“它们需要什么 discovery infrastructure？”** ([source](https://x.com/omooretweets/status/2049516859658293303))

5. **Feedback-to-skill 缺口成为当天最一致的从业者挫败感：agents 接受纠正，但不会把纠正 codify 成 reusable skills，开发者被迫在 sessions 间反复纠正同样错误。** 当前 workarounds（memories、manual skills、cronjobs）都被描述为不足。([source](https://x.com/RhysSullivan/status/2049540492908810559))

6. **Martin Fowler 对 verification-first harness engineering 的认可——“the game is not 'how fast can we build' but 'how fast can we tell whether this is right'”——说明该概念已被软件架构主流接受，完成从 Twitter thread 到 Fowler recommendation 的转变。** ([source](https://x.com/martinfowler/status/2049480328423608522))
