## 2. 令人困扰的问题

### 智能体自我修改限制显得武断 -- Severity: Medium

[@thsottiaux](https://x.com/thsottiaux) [解释](https://x.com/thsottiaux/status/2048114254465921366)（546 次点赞），出于安全考虑，不允许 agents 编辑自己的权限设置。[@nicdunz](https://x.com/nicdunz) 带着挫败感回复：“你能允许它这么做，然后只告诉它不要这么做吗。”一条解释限制的帖子拿到 546 次点赞，说明有大量受众在关注权限边界。开发者想要更细粒度地控制 agents 能修改什么、不能修改什么，而不是一刀切限制。

Prevalence: Active -- 这是 safety-by-default 和 developer autonomy 之间反复出现的张力。

### 实时部署智能体前缺少回测 -- Severity: Medium

[@Vladii_x](https://x.com/Vladii_x) [评测](https://x.com/Vladii_x/status/2047979576597344436)（7 次点赞）Chance AI 的 no-code trading agent：“有一个 ‘but’：希望在真实资金前有 test mode。我对自己的 trading idea 没那么有信心。先 backtest。再 deposit。”[@bonduelleioat](https://x.com/bonduelleioat) 表示同意。这反映了更广泛的模式：智能体平台先发布执行能力，再补验证能力，把风险推给用户。

Prevalence: Emerging -- 随着 no-code agent builders 增多，缺少模拟/测试模式会成为常见缺口。

### 编程智能体质量与定价张力仍在 -- Severity: Medium

[@burkov](https://x.com/burkov) [认为](https://x.com/burkov/status/2047835447976431829)（79 次点赞）“a coding agent harness is now open source”，因此 Cursor $60B 的估值站不住脚。[@DAIEvolutionHub](https://x.com/DAIEvolutionHub) [指出](https://x.com/DAIEvolutionHub/status/2048077092785676452)，Claude Code 从 $20 方案中移除后，开发者把它重建成开源 OpenClaude。挫败感是结构性的：当开源替代品能在几天内追上，付费工具就必须证明自己的溢价合理。

Prevalence: Recurring -- 这在 4 月 23 日和 24 日也出现过。每次价格调整或限制都会产生开源回应。

### 链上治理失败助长 Exploits -- Severity: High

[@CerbAgent](https://x.com/CerbAgent) [分析](https://x.com/CerbAgent/status/2048107305149899112)（32 次点赞）Purrlend $1.5M drain：“2/3 multisig，没有 timelock，攻击者在仅 8 小时前被添加为 ‘bridge’。”[@icodeforlove](https://x.com/icodeforlove) 回复：“2/3 multisig + no timelock 基本就是一台被攻陷的笔记本加一杯咖啡休息时间。8 小时 dwell time 才是关键细节。”这次 exploit 是治理失败，而不是技术复杂性问题，而且没有 monitoring agent 及时标记新 signer。

Prevalence: Active -- DeFi protocols 中的治理失败仍是主要攻击向量，agent-based monitoring 已被提出，但尚未成为标准。

## 3. 人们期望的功能

### 智能体模拟和回测环境

多条帖子显示，agents 正在没有 dry-run 能力的情况下进入生产。[@Vladii_x](https://x.com/Vladii_x/status/2047979576597344436) 明确要求在真实资金前加入 backtest mode。[@kwindla](https://x.com/kwindla) [指出](https://x.com/kwindla/status/2048123872948670560)，构建面向用户的 AI 应用是“一项全栈软件工程活动”，需要从模型到 UX 的完整闭环。目前没有标准 agent testing harness 能在金融、语音或浏览器自动化任务中提供模拟。

Urgency: High -- Opportunity: [++]

### 智能 Skill 发现与加载

skills 生态继续扩张——[@aiedge_](https://x.com/aiedge_) [整理](https://x.com/aiedge_/status/2047978980155019452)（9 次收藏）了“Google、Notion、Figma、Canva 官方 AI skills 的完整库”。[@tom_doerr](https://x.com/tom_doerr) 分别为 [local-first agent frameworks](https://x.com/tom_doerr/status/2047921691565625349) 和 [Manim animations](https://x.com/tom_doerr/status/2047976424770764879) 发布 skills。但发现问题仍然存在：没有 marketplace 提供质量评分和 context-aware skill selection。4 月 24 日关于 attention dilution 的研究（42% budget 被无关 skills 吸收）让这个问题变得紧迫。

Urgency: High -- Opportunity: [+++]

### 实时链上智能体监控

[@CerbAgent](https://x.com/CerbAgent/status/2048107305149899112) 提出了 Purrlend exploit 当时所需的解决方案：“当 bridge role 被分配给未知地址时触发警报。pre-transaction scan 阻止第一次 withdrawal attempt。auto-revoke 在资金移动前剥离权限。”[@irdh34](https://x.com/irdh34/status/2048009137468985637) 报告 HiveFury 已在 wallet protection 上运行，但这个类别缺少标准化。每个 DeFi protocol 都独自实现（或不实现）监控。

Urgency: High -- Opportunity: [++]

### 不发生 Context Rot 的持久智能体记忆

[@asim_Ai1](https://x.com/asim_Ai1) [描述](https://x.com/asim_Ai1/status/2048070207873397133) 自己在 Obsidian 中构建“一个第二大脑，我用的每个 AI agent 都能自动读取、更新和学习”。[@mstockton](https://x.com/mstockton/status/2048115926038716925) 建议“把 sessions 提炼成 agent 可以读取的 markdown”。这些都是手动权宜方案，用来弥补原生跨 session memory 的缺失；理想状态是记忆会随时间累积，而不会退化。

Urgency: Medium -- Opportunity: [++]

## 7. 机会在哪里

**[+++]** **Agent skill discovery and quality scoring** -- skills 生态继续扩张（Ritual testnet skills、image-taste-mobile、Manim skills、Byreal agent skills），但没有 quality gate 或智能加载机制。Laureum AI 评分 28 个 MCP servers，发现 process quality 平均只有 55.5/100。没有 marketplace 能把 skill quality scores 与 context-aware loading 连接起来，防止 attention dilution。来源：[@cryptodeadline](https://x.com/cryptodeadline/status/2047942019431989551), [@aiedge_](https://x.com/aiedge_/status/2047978980155019452), [@blueemi99](https://x.com/blueemi99/status/2047816714767757430)。

**[+++]** **Agent-to-agent commerce infrastructure** -- Anthropic 的 Project Deal 展示了 186 笔真实交易（$4,000+），且结果受模型 tier 影响。当前没有面向 agent-to-agent negotiation、settlement 或 dispute resolution 的标准协议。金融板块影响（illiquid asset desks、distressed claims）已经很具体。来源：[@VaibhavSisinty](https://x.com/VaibhavSisinty/status/2047923077481365979), [@JamesonCamp](https://x.com/JamesonCamp/status/2047747834079007117), [@TechCrunch](https://x.com/TechCrunch/status/2048156238614274272)。

**[++]** **Managed agent infrastructure at commodity pricing** -- Claude Managed Agents 每 session-hour $0.08，KiloClaw 60 秒部署，OpenClaw 语音交接都表明 agent deployment 正在变成基础设施，而不是工程项目。缺口在 vertical specialization：为特定行业（法律、医疗、金融）调优，并内置合规能力的 managed agents。来源：[@coreyganim](https://x.com/coreyganim/status/2048090410757890308), [@Rixhabh__](https://x.com/Rixhabh__/status/2047980397674574332)。

**[++]** **Real-time on-chain security agents** -- Purrlend $1.5M exploit（治理失败、8 小时 dwell time）和 HiveFury 的 wallet sentinel 同时展示了问题和早期解决方案。跨 DeFi protocols 监控权限、approvals 和资金流动的标准化 monitoring agents，还没有作为基础设施出现。来源：[@CerbAgent](https://x.com/CerbAgent/status/2048107305149899112), [@irdh34](https://x.com/irdh34/status/2048009137468985637)。

**[+]** **Agent simulation and backtesting platforms** -- 随着 no-code agent builders 增多（Chance AI、PlutonAI），缺少 dry-run environments 会把风险推给终端用户。支持金融、语音和浏览器自动化场景的标准化 agent testing harness，可以同时服务开发者和平台。来源：[@Vladii_x](https://x.com/Vladii_x/status/2047979576597344436), [@kwindla](https://x.com/kwindla/status/2048123872948670560)。
