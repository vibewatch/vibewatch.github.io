## 2. Pain Points: What Frustrates People

**Severity: Moderate | Prevalence: High**

Despite three consecutive days of harness engineering dominance, the field lacks standardized practices. [@drummatick](https://x.com/drummatick) published a curriculum for "top 1% AI engineer" listing agent harness, memory, context engineering, and subagent spawning as required competencies — but these are still learned through X threads and individual experimentation, not structured courses (258 likes, 331 bookmarks, [tweet](https://x.com/drummatick/status/2042553870191333517)). ### Vault-Level Security Gaps Persist
**Severity: High | Prevalence: Low (enterprise early adopters)**

Coping: avoiding vaults for sensitive credentials, using external secret managers.

### Skills Discovery Gap Widens
**Severity: Moderate | Prevalence: Moderate**

The Swift-Agent-Skills directory (1.2K stars, 14 categories) and Hermes skills browsing address iOS and Hermes ecosystems respectively, but no universal skills discovery mechanism exists. Each framework builds its own silo. autoskills CLI solves installation but not discovery. Coping: curated lists on GitHub, manual sharing on X.

### Stateful Agent Complexity
**Severity: High | Prevalence: Moderate**

[@diptanu](https://x.com/diptanu) argued that stateful sandboxes reduce 70–80% of the complexity in agentic applications, but current solutions require significant infrastructure expertise (55 likes, 35 bookmarks, [tweet](https://x.com/diptanu/status/2042935455310455130)).
