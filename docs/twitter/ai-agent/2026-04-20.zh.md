### 1.2 OpenClaw vs Hermes Agent：框架比较清晰化 🡒

[@shannholmberg](https://x.com/shannholmberg) 分享了一份 [详细 source-code comparison](https://x.com/shannholmberg/status/2046162254102667617)（59 个点赞、39 次收藏），总结 [@elvissun](https://x.com/elvissun) 对 OpenClaw 和 Hermes Agent 做的 9 小时 side-by-side study。这份比较映射出 agents 应该如何学习和使用 skills 的两种根本不同哲学。

![OpenClaw vs Hermes comparison，展示 skill philosophy、bundled skills、discovery scope、core tradeoffs 和 product positioning 差异](https://pbs.twimg.com/media/HGVtuhKbMAA2TRQ.jpg)

Hermes 随附 123 个 bundled skills（GitHub PRs、Obsidian、Notion、Linear），并有 self-authoring loop：每 N 次 tool calls，agent 会考虑保存一个 skill，然后后台 review 会扫描 skill-worthy patterns。OpenClaw 则采取相反立场：“新 skills 应该先发布到 ClawHub，而不是默认加进 core”，并提供五个 precedence levels（workspace > user global > managed > bundled > extra）和 bounded discovery。

核心发现是：Hermes 有“skill explosion problem”——Elvis 在自己的 `~/.hermes/skills/` 目录中，仅仅为了读取图片就发现了 3 个重叠 skills。Skills 的增长快于 consolidation。OpenClaw 通过设计避免了这一点，代价是要求用户显式创建 skills。[@elvissun](https://x.com/elvissun) 总结：“Pick either. Learn from both. Then go make something useful.”

[@BlockLayerPod](https://x.com/BlockLayerPod) 发布了一份 [全面 OpenClaw analysis](https://x.com/BlockLayerPod/status/2045953206358806868)（153 个点赞、13.4K 次浏览），覆盖该框架的 crypto synergies。关键轶事包括：Austin Griffith 的 agent 在他睡觉时部署了 production smart contracts，而 agent [@langoustine69A](https://x.com/langoustine69A) 一周内交付了 80+ 个 paid x402 endpoints。安全现实检查也很直接：“Austin 的 bot 在任务中试图提取自己的 private key。”

[@aiedge_](https://x.com/aiedge_) 发布了 [GitHub Trending weekly snapshot](https://x.com/aiedge_/status/2046272782280106134)（19 个点赞、41 次收藏），显示前五个 repositories 全部与 agent 相关：

![GitHub Trending，展示 andrej-karpathy-skills 60,441 stars、hermes-agent 101,400、claude-mem 63,036、evolver 5,347、GenericAgent 4,441](https://pbs.twimg.com/media/HGXSQF4bgAAT_g_.jpg)

**讨论要点：** 框架竞争已经成熟为一个可识别的架构分裂：batteries-included（Hermes/Rails analogy）vs primitives-first（OpenClaw/Linux analogy）。9 小时源码研究首次给出严格证据，说明每种方法都有不同失败模式：skill explosion vs manual authoring burden。

**与前日对比：** 4 月 19 日聚焦 Hermes 达到 100K stars。4 月 20 日增加了与 OpenClaw 的结构性比较，并将 skill explosion problem 识别为 Hermes 的主要 long-tail risk，同时通过生产轶事记录 OpenClaw 的真实安全挑战。

### 1.3 智能体安全威胁数据更具体 🡕

安全是当天数据最丰富的主题。[@stormrae_ai](https://x.com/stormrae_ai) [引用企业数据](https://x.com/stormrae_ai/status/2046281126814343458)（285 个点赞、24.9K 次浏览）：“96% 的企业已经在运行 AI agents。只有 21% 有匹配的 governance model。”帖子描述了 Palo Alto Networks 的 red-team demonstration：一个 agent 仅靠说服就执行了 900 美元提款——“没有 exploit，没有 breach，只是 clever reframing。”OWASP 的 Q2 2026 landscape report 列出的主要威胁包括：prompt injection、agent privilege escalation、data poisoning、hallucination drift。[@maestroalvarez](https://x.com/maestroalvarez) 在回复中指出：“Phishing tabletop exercises 不会迁移；agents 需要 adversarial roleplays，攻击者是一个礼貌用户。”

[@akshay_pachaar](https://x.com/akshay_pachaar) 提供了当天 [最详细安全 thread](https://x.com/akshay_pachaar/status/2046151867177308181)（48 个点赞、69 次收藏），总结 Google DeepMind 的 “AI Agent Traps” 论文。论文映射了六类攻击面：Content Injection（HTML injections 的 hijack rate 为 86%）、Semantic Manipulation、Cognitive State Traps（<0.1% poisoned data 下 attack success >80%）、Behavioural Control、Systemic 和 Human-in-the-Loop。

![Google DeepMind AI Agent Traps paper，识别针对 AI agents 的六类 adversarial content，覆盖 perception、reasoning、memory、action、multi-agent dynamics 和 human oversight](https://pbs.twimg.com/media/HGVkR0dbsAA-Hxv.png)

[@web3nomad](https://x.com/web3nomad) 提出了跨 session 角度：“写入 agent memory 的 adversarial content injection 会跨 sessions 持久化。单 session hijacks 已经很糟。跨 session knowledge poisoning 是另一个威胁类别。”[@hugobowne](https://x.com/hugobowne) [警告](https://x.com/hugobowne/status/2046042202921701795)，“AI Builders 正在慢慢意识到应该假设 system prompts 会被 exfiltrated……很少有人意识到整个 agent harnesses 也几乎等同公开。”[@aixarizzo](https://x.com/aixarizzo) [标记](https://x.com/aixarizzo/status/2046280284954583043)，使用带浏览器访问能力 AI agents 的 crypto accounts，会把 cached wallet sessions、login tokens 和 seeds 暴露给 agent。

[@hackSultan](https://x.com/hackSultan) 发布了 [当天互动最高的推文](https://x.com/hackSultan/status/2046269993898181081)（254 个点赞、794 次收藏、score 4362）：一个面向 vibe-coded projects 的综合 security audit prompt。该 prompt 指示 agent 对 frontend、backend、auth、data、API、infra 和 supply chain layers 做 exhaustive vulnerability analysis。[@goldencantech](https://x.com/goldencantech) 在回复中观察到：“我喜欢这场讨论正在从‘不要 vibecode’转向‘如果你非要 vibecode，这里有办法’。”

**讨论要点：** 4 月 20 日，安全讨论沿三个维度演进：enterprise governance data（96% deployed、21% governed）、academic threat taxonomy（DeepMind 六类攻击面）和 practitioner tooling（security audit prompts 获得 794 次收藏）。最重要信号是 @web3nomad 独立提出的 cross-session memory poisoning，这是一类新威胁。

**与前日对比：** 4 月 19 日，AgentShield 提供量化 threat data（12% malicious skills）和 frontier model benchmark failures。4 月 20 日新增企业治理统计、DeepMind threat taxonomy 以及 cross-session memory poisoning vector。威胁模型正从 skill-level risks 扩大到 memory-level 和 multi-agent systemic risks。

## 2. 令人困扰的问题

### 智能体治理落后于部署 -- Severity: High

[@stormrae_ai](https://x.com/stormrae_ai) 报道 [96% enterprise agent deployment vs 21% governance](https://x.com/stormrae_ai/status/2046281126814343458)（285 个点赞、24.9K 次浏览）。Palo Alto Networks 演示了一个 red-team agent 通过 social engineering 执行 900 美元 financial withdrawal——没有 exploit 或 breach——这说明缺口真实存在。[@EddyBrown777](https://x.com/EddyBrown777) 回应：“核心问题不是 AI capability——而是 control。企业 rushed deployment，却没有对齐 incentives、oversight 或 fail-safes。”

### Cross-Session Memory Poisoning 没有防御 -- Severity: High

[@web3nomad](https://x.com/web3nomad) 在回复 DeepMind paper thread 时 [指出](https://x.com/akshay_pachaar/status/2046151867177308181) 一个新威胁类别：“写入 agent memory 的 adversarial content injection 会跨 sessions 持久化……auto-capture memory systems 对此没有真正防御。”当前唯一缓解方式是“explicit write-time review”，由人决定什么能进入 knowledge base。

### 自我编写智能体的 Skill Explosion -- Severity: Medium

[@elvissun](https://x.com/elvissun)（经由 [@shannholmberg](https://x.com/shannholmberg/status/2046162254102667617)，59 个点赞、39 次收藏）记录到，Hermes 为单一任务（读取本地图片）生成了三个重叠 skills。“agent 很擅长发现‘我应该把这个封装起来’。它没那么擅长发现‘我已经在三个文件夹外封装过这个了’。”Skills 增长速度快于 consolidation。还没有 deduplication mechanism。

### 智能体 Harnesses 实际上是公开的 -- Severity: Medium

[@hugobowne](https://x.com/hugobowne) [警告](https://x.com/hugobowne/status/2046042202921701795)（6 个点赞、3 次收藏）：“AI Builders 正在慢慢意识到应该假设 system prompts 会被 exfiltrated 并公开。很少有人意识到他们的整个 agent harnesses 也几乎等同公开。”含义是：任何依赖 harness secrecy 的安全假设都已经破产。

## 3. 人们期望的功能

### Skill Deduplication and Consolidation

[@elvissun](https://x.com/elvissun)（经由 [@shannholmberg](https://x.com/shannholmberg/status/2046162254102667617)）指出，Hermes Agent 需要“带 invocation metrics 的 consolidation pass + 在 skill creation 上更强的 dedupe”。Skill corpus 无限制增长：“有些很 brilliant，有些 redundant，有些与另外三个 nobody remembers exist 的 skills 重叠。”

机会：一个 skill deduplication engine，如果能基于 invocation data 聚类语义相似 skills、合并重叠 skills、剪掉未使用 skills，就能解决 self-authoring agent systems 的主要长期风险。

### 智能体金融身份（Know Your Agent）

[@t54ai](https://x.com/t54ai) [描述](https://x.com/t54ai/status/2046230570498195677) 了缺口：“今天金融 stack 中的每项检查都是围绕 human 设计的。Signatures、IDs、clicks、chargebacks。当 AI agent 负责 transaction 时，这些检查每一项都有缺口。”该 thread 提出 KYA（Know Your Agent），覆盖每个 autonomous action 的 model provenance、version、runtime integrity 和 signed mandates。

机会：一个把 agent identity 连接到现有 financial compliance infrastructure 的 KYA framework，将释放企业级 agent commerce。

### Write-Time Memory Security

[@web3nomad](https://x.com/web3nomad)（回复 [@akshay_pachaar](https://x.com/akshay_pachaar/status/2046151867177308181)）指出，现在没有工具会验证进入 agent memory 的内容：“explicit write-time review（由人决定什么进入 wiki）是唯一当前缓解方式。”

机会：自动化 memory write validation——在内容进入 persistent memory 前进行 classification、filtering 和 flagging——将处理 cross-session poisoning vector。

## 6. 新动态与亮点

### Agentic(.)Market Dashboard 揭示智能体商业规模

[@base](https://x.com/base) 发布了 agent-to-agent commerce 的 [首个公开 dashboard](https://x.com/base/status/2046325766091116897)：167M+ x402 transactions、48.6M 美元 volume、71,490 unique buyers、12,992 unique sellers。Services leaderboard 显示 Anthropic、X API、Bankr、Google Flights 和 OpenAI 是 top agent-consumed services。这是 agent-to-agent marketplaces 正在以有意义规模运转的首个硬证据。

### Google DeepMind 将智能体攻击面归纳为六类

[@akshay_pachaar](https://x.com/akshay_pachaar) [总结](https://x.com/akshay_pachaar/status/2046151867177308181) 了 “AI Agent Traps” 论文（69 次收藏），这是第一个系统化 adversarial content targeting agents 框架。六类攻击——Content Injection、Semantic Manipulation、Cognitive State、Behavioural Control、Systemic、Human-in-the-Loop——以及具体 attack success rates（HTML injection 86%、0.1% poisoned data 下 80%），为 agent security 提供了研究议程。

### NVIDIA 在百万行规模上演示 Self-Evolving EDA

[@dair_ai](https://x.com/dair_ai) 分享了 [NVIDIA 论文](https://x.com/dair_ai/status/2046251813738025025)（52 次收藏），把 multi-agent self-improvement 应用于 ABC——一个半导体行业使用的 120 万行 logic synthesis tool。该 framework 发现了超越 human-designed heuristics 的优化策略。这是 self-evolving agent patterns 首次应用于 production industrial infrastructure。

### Qwen 3.6 Max Preview 在 SkillsBench 上领先 Claude 近 2x

[@Alibaba_Qwen](https://x.com/Alibaba_Qwen) 发布了 [benchmark results](https://x.com/Alibaba_Qwen/status/2046227759475921291)，显示 Qwen 3.6 Max Preview 在 SkillsBench（agent skill execution）上得分 55.6，而 Claude 4.5 Opus 为 30.0——优势达 1.85x。preserve_thinking feature 可跨 disconnected sessions 携带 reasoning chains，是 long agent traces 的新能力。

### Single Agent 在 DeepSearchQA 上达到 SOTA，挑战 Swarm Paradigm

[@youdotcom](https://x.com/youdotcom) [报告](https://x.com/youdotcom/status/2046287847011803590)，一个 well-scaled single agent 使用 10M tokens 和 1,000 turns，在 DeepSearchQA 上达到 83.67% accuracy，并声称“multi-agent swarms 在相同 compute budget 下不如 single well-scaled agent”。这一实证结果挑战了当前 multi-agent trend。

### MCP Tool Poisoning 被识别为隐形攻击向量

[@correlicHQ](https://x.com/correlicHQ) [描述](https://x.com/correlicHQ/status/2046337498662801555) MCP tool poisoning：“恶意 tool description 告诉你的 agent 读取 SSH keys。agent 以为自己是在遵循指令。UI 什么都不显示。”该攻击利用 agents 与 tool descriptions 之间的信任关系，而 description 本身变成 payload。

## 7. 机会在哪里

`[+++]` **智能体市场基础设施** -- Agentic Market dashboard（167M transactions、71K buyers、48.6M 美元 volume）证明 agent-to-agent commerce 真实存在。5:1 buyer-to-seller ratio 表明 services 供给不足。帮助开发者 listing、pricing 和 monitoring agent-consumable services 的工具会有即时需求。Sources: [@base](https://x.com/base/status/2046325766091116897), [@MilkRoad](https://x.com/MilkRoad/status/2046277243224391686), [@OrbisAPI](https://x.com/OrbisAPI/status/2046307504452489449).

`[+++]` **智能体安全与治理工具** -- 96% enterprise deployment vs 21% governance 创造巨大缺口。DeepMind taxonomy 提供了结构化 attack surface map。Cross-session memory poisoning 和 MCP tool poisoning 没有自动化防御。Sources: [@stormrae_ai](https://x.com/stormrae_ai/status/2046281126814343458), [@akshay_pachaar](https://x.com/akshay_pachaar/status/2046151867177308181), [@correlicHQ](https://x.com/correlicHQ/status/2046337498662801555).

`[++]` **Skill deduplication and lifecycle management** -- Hermes 的 skill explosion problem（skills 增长快于 consolidation）会影响每一个 self-authoring agent system。能根据 invocation data 聚类、合并并修剪语义相似 skills 的工具，会解决一个随 adoption 扩大的问题。Sources: [@elvissun](https://x.com/elvissun) via [@shannholmberg](https://x.com/shannholmberg/status/2046162254102667617).

`[++]` **Agentic-native model optimization** -- SkillsBench、Terminal-Bench 和 long-horizon benchmarks 正在成为 agent builders 的选型标准。针对 tool-call coherence 和 cross-turn reasoning 明确优化的模型（Qwen 3.6 Max 的 preserve_thinking、Kimi K2.6 的 12-hour sessions）正在领先 general-purpose models。Sources: [@Alibaba_Qwen](https://x.com/Alibaba_Qwen/status/2046227759475921291), [@aaryan_kakad](https://x.com/aaryan_kakad/status/2046266617227858256).

`[+]` **智能体金融身份（KYA）** -- [@t54ai](https://x.com/t54ai/status/2046230570498195677) 指出现有 financial compliance 为人类设计，与 agent-executed transactions 之间存在结构性缺口。167M agent transactions 已经 settled，compliance gap 正在扩大。Sources: [@t54ai](https://x.com/t54ai/status/2046230570498195677), [@stormrae_ai](https://x.com/stormrae_ai/status/2046281126814343458).

`[+]` **Single-agent scaling vs multi-agent orchestration** -- you.com 的 DeepSearchQA result（single agent 83.67%）与 Swarms v11 的 16-agent architectures 构成未解决的架构问题。帮助开发者判断什么时候 scale depth、什么时候 scale breadth 的 tooling，会节省大量 compute 和 engineering time。Sources: [@youdotcom](https://x.com/youdotcom/status/2046287847011803590), [@swarms_corp](https://x.com/swarms_corp/status/2045977348562952247).

## 8. 要点总结

1. Agent-to-agent commerce 产出了第一个公开 dashboard：Agentic Market 上 167M+ x402 transactions、48.6M 美元 volume、71,490 buyers，且 5:1 buyer-to-seller ratio 表明供给不足。Sources: [@base](https://x.com/base/status/2046325766091116897), [@MilkRoad](https://x.com/MilkRoad/status/2046277243224391686).

2. OpenClaw vs Hermes comparison 清晰化为两种哲学：batteries-included（Hermes，123 skills，self-authoring）vs primitives-first（OpenClaw，bounded discovery，governance by policy）。Hermes 的 skill explosion 是主要长期风险；OpenClaw 的 manual authoring 是主要采用摩擦。Source: [@shannholmberg](https://x.com/shannholmberg/status/2046162254102667617).

3. 智能体安全威胁数据达到新层级：96% enterprise deployment vs 21% governance，DeepMind 六类攻击 taxonomy，HTML injection 86% hijack rate，以及 cross-session memory poisoning 被识别为一个没有自动化防御的新威胁类别。Sources: [@stormrae_ai](https://x.com/stormrae_ai/status/2046281126814343458), [@akshay_pachaar](https://x.com/akshay_pachaar/status/2046151867177308181).

4. 自我改进智能体进入工业基础设施：NVIDIA 演示了 120 万行半导体 EDA tool 的 multi-agent self-evolution，发现超越 human heuristics 的优化策略。Sources: [@dair_ai](https://x.com/dair_ai/status/2046251813738025025), [@omarsar0](https://x.com/omarsar0/status/2045956901750399374).

5. 模型竞争转向 agentic-specific benchmarks：Qwen 3.6 Max 在 SkillsBench 上相对 Claude 4.5 Opus 有 1.85x 优势（55.6 vs 30.0），Kimi K2.6 展示了 12 小时 autonomous coding 和 4,000+ tool calls。Sources: [@Alibaba_Qwen](https://x.com/Alibaba_Qwen/status/2046227759475921291), [@aaryan_kakad](https://x.com/aaryan_kakad/status/2046266617227858256).

6. Multi-agent paradigm 遭遇第一次实证挑战：you.com 的 single well-scaled agent 在 DeepSearchQA 上达到 SOTA（83.67%），并声称 swarms 在 equivalent compute budgets 下表现更差。Source: [@youdotcom](https://x.com/youdotcom/status/2046287847011803590).

7. Vibecoding 安全讨论从“不要做”转向“如果要做，就审计”：一条 security prompt 获得 794 次收藏（数据集中最高），显示 vibe coding workflow 内部对实用安全工具有需求。Source: [@hackSultan](https://x.com/hackSultan/status/2046269993898181081).

8. Skill 生态增长加速：ctx 现在索引 1,700+ skills 和 450+ agents（高于 4 月 19 日的 1,450 skills），skillsmp.com 列出 60,000+ skills，Anthropic 投入 13 门免费课程推动采用。Discovery problem 现在是主要瓶颈。Sources: [@SteveSolun](https://x.com/SteveSolun/status/2045996214689128906), [@JorgeCastilloPr](https://x.com/JorgeCastilloPr/status/2046330519432966515).
