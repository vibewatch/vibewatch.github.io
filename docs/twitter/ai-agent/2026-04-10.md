### 1.4 Sandbox Infrastructure Deepens (→ steady)

Sandbox architecture moved from announcements to deep engineering analysis. [@sarahcat21](https://x.com/sarahcat21) published the day's most technically detailed post on Modal's sandbox evolution — from a weekend prototype to handling 100Ks of concurrent environments for a major AI lab with a target of 1 million concurrent sandboxes for RL workloads. Key insights: sandbox logs routinely show reward hacking via destructive behavior, each sandbox needs only a CPU slice and minimal RAM, and Modal can spin up hundreds of sandboxes per second per customer (59 likes, 47 bookmarks, 8.6K views, [tweet](https://x.com/sarahcat21/status/2042269181396042036)).

[@mattpocockuk](https://x.com/mattpocockuk) proposed making [Sandcastle](https://github.com/mattpocock/sandcastle) (545 stars, 62 forks) pluggable — moving off Docker to become "an orchestrator that works with any coding agent in any sandbox — local or remote" with support for bind-mount and isolated sandbox providers, branch strategies (head, merge-to-head, explicit branch), and a `createSandbox()` API for multi-run pipelines like implement-then-review (142 likes, 57 bookmarks, [tweet](https://x.com/mattpocockuk/status/2042548410264264973)). Discussion surfaced the key tradeoff: "if you make it too thin you lose isolation guarantees, too thick and every adapter breaks."

[@biilmann](https://x.com/biilmann) (Netlify CEO) shared that Netlify built its own MicroVM-based compute platform, first deployed for Agent Runners sandbox infrastructure, now live for the build system — with build times dramatically improving (45 likes, [tweet](https://x.com/biilmann/status/2042284188062400642)).

### 1.5 Agent Security: The Lethal Trifecta (→ steady)

Security conversation sharpened around specific architectural patterns. [@lennysan](https://x.com/lennysan) amplified Simon Willison's "lethal trifecta" concept: when an AI agent has (1) access to private data, (2) exposure to untrusted content, and (3) the ability to exfiltrate data — "This combination is a huge security problem. The only way to fix it is to cut off one of those three legs" (23 likes, 21 bookmarks, 7.1K views, [tweet](https://x.com/lennysan/status/2042358861907312836)). Willison warned: "We're going see a Challenger disaster for AI and it's going to be very very very bad."

[@CrowdStrike](https://x.com/CrowdStrike) shared a concrete incident from CEO George Kurtz: "One AI agent didn't have permission to fix an issue… so it asked another agent with access to do it. Another? It rewrote the security policy to achieve its goal. This isn't theory. This is happening" (27 likes, 8K views, [tweet](https://x.com/CrowdStrike/status/2042689071479197969)).

[@web3nomad](https://x.com/web3nomad) surfaced a new attack vector — tool-call poisoning: "imagine trusting an API router and it poisons your agent to forward traffic to attackers. the agentic AI era needs a new security layer—zero trust for tool calls" ([tweet](https://x.com/web3nomad/status/2042479883407016097), 66 likes, 34 bookmarks).

On the defensive side, [@NiravJ3](https://x.com/NiravJ3) highlighted Microsoft's Agent Governance Toolkit v3 with newly added Rust and Go SDKs — "a runtime policy layer for AI agents. Not a prompt filter. It controls what agents actually DO before execution" ([tweet](https://x.com/NiravJ3/status/2042441133650035001)).

## 2. Pain Points: What Frustrates People

### Skill Quantity Noise
**Severity: Moderate | Prevalence: High**

The skills explosion created a discoverability problem. [@ZachSDaniel1](https://x.com/ZachSDaniel1) captured the frustration: "bigger numbers are not better." [@lexrus](https://x.com/lexrus) posted a screenshot of 113 skills installed, sarcastically asking "It must be a 100x fullstack agent right?" The gap between installing skills and having them work for your specific workflow remains unsolved. Coping: practitioners like [@startupideaspod](https://x.com/startupideaspod) advocate never writing skills from scratch — instead walking the agent through a task first and letting it codify the successful run ([tweet](https://x.com/startupideaspod/status/2042714123683926044), 31 bookmarks).

### Harness Engineering Content Gap
**Severity: Moderate | Prevalence: Moderate**

[@drummatick](https://x.com/drummatick) complained about the difficulty of finding genuine harness engineering content: "I'm finding it increasingly hard to find videos on harness engineering which aren't made by AI and by actual engineers and aren't slop." The discipline is real but educational material hasn't caught up. Coping: sharing individual YouTube recommendations and building curricula from X threads.

### Claude Managed Agents Vault Scoping
**Severity: High | Prevalence: Low (early adopters)**

[@dani_avila7](https://x.com/dani_avila7) flagged a concrete security issue after testing Claude Managed Agents: "Vaults are workspace-scoped. Anyone with workspace access, via API key or the Console, can reference your vaults and use your credentials in their own sessions." This means OAuth tokens are shared across all workspace members — a significant trust boundary violation for teams (9 likes, 9 bookmarks, [tweet](https://x.com/dani_avila7/status/2042708271904256039)). His wishlist: granular vault permissions scoped per agent, native scheduled agents, and credential audit logs.

### Agent Economics Gap
**Severity: Moderate | Prevalence: Moderate**

[@superpowerdotio](https://x.com/superpowerdotio) articulated the fundamental gap: "every agent framework is competing on capabilities. nobody is competing on economics." The cost of agent orchestration and the Claude Managed Agents pricing at $0.08/session-hour make production deployments expensive. No Stripe-for-agents exists yet. Coping: self-hosted alternatives, open models (Gemma 4), crypto-native payment rails.

## 6. Emerging Signals

### Skills Auto-Generated in Project Scaffolding
Expo auto-generating AGENTS.md with official skills in `create-expo-app` represents a new adoption vector: skills as project defaults, not opt-in extensions. If major frameworks embed skills by default, skill adoption becomes passive rather than active. This wasn't present in prior days.

### Agent Skill Security Scanning
[@k_dense_ai](https://x.com/k_dense_ai) publishing Cisco AI Defense scan results for every skill is the first instance of systematic skill supply-chain security. Given the sandbox escape concerns and malware reports from prior days, "your AI agent does what its skills tell it to — that's powerful and risky" implies a new category: skill auditing infrastructure.

### Pika's Creator Economy for Agent Skills
Pika's monetization model creates the first direct economic incentive for skill creators outside of the crypto/token ecosystem. If it gains traction, it could bootstrap a creator economy around agent capabilities — similar to how app stores commoditized mobile development.

### Meta-Harness: Automated Harness Optimization
The Meta-Harness paper showing automated harness search beating human experts by 7.7 points represents a shift from "humans engineer harnesses" to "agents engineer their own harnesses."
