## 2. Pain Points: What Frustrates People

### 🔴 Agent Marketplace Security Still Critical
**Description:** The 12% malware stat from 04-07 continues circulating — [@FOUNDATIONdvcs](https://x.com/FOUNDATIONdvcs) (score 33.4) [lists it](https://x.com/FOUNDATIONdvcs/status/2041927970684600331) alongside $480M stolen from crypto users and 676M SSNs leaked.
**Scenario:** Developer installs a skill from a marketplace; it exfiltrates credentials already granted to the agent.
**Severity:** 🔴 Critical — unchanged from prior days.
**Prevalence:** Persistent across all three days. [@FangcunLeap](https://x.com/FangcunLeap) (score 12.7) [built Fangcun SkillWard](https://x.com/FangcunLeap/status/2042264140346167452), an open-source security scanner for AI Agent Skills.
**Coping strategies:** SkillWard scanner, scoped OAuth, manual code review, `--dry-run` flags (autoskills), Swarms' automated validation layer.

### 🔴 Managed Agent Pricing vs Value
**Description:** Claude Managed Agents API pricing is ~18× less efficient than Max subscriptions. [@MLStreetTalk](https://x.com/MLStreetTalk) (score 37.5) [calculates](https://x.com/MLStreetTalk/status/2042085356909293726) >$3,000/month vs $200/month for equivalent workloads.
**Scenario:** Team evaluates Managed Agents for production; discovers that trivial email triage costs hundreds per month at API rates.
**Severity:** 🔴 Critical — blocks adoption for cost-sensitive teams.
**Prevalence:** New pain point on 04-09, emerging from 04-08's launch excitement.
**Coping strategies:** Self-host Claude Code containers on AWS using Max subscription keys; use for high-value workflows only.

### 🟡 Harness Engineering Content Quality
**Description:** [@drummatick](https://x.com/drummatick) (score 28.2) [complains](https://x.com/drummatick/status/2042291393167835473): "I'm finding it increasingly hard to find videos on harness engineering which aren't made by AI and by actual engineers and aren't slop."

**Prevalence:** Multiple users ([@drummatick](https://x.com/drummatick), [@BhosalePratim](https://x.com/BhosalePratim)) expressing this.
**Coping strategies:** [@drummatick](https://x.com/drummatick) (score 25.1) [eventually found](https://x.com/drummatick/status/2042296679999926423) a good video and shared it.

### 🟡 Hermes Agent UX Gaps
**Description:** [@conoro](https://x.com/conoro) (score 27.7) [reports](https://x.com/conoro/status/2042098373826593154): "I asked it to build a simple scraper, which it did. By dumping a pile of files/code into its own hermes-agent folder! It doesn't seem to be aware of any framework it should use to create Agents."
**Scenario:** New user expects Hermes to behave like OpenClaw but gets unstructured file dumps instead.
**Severity:** 🟡 Moderate — fixable with better defaults.
**Prevalence:** [@conoro](https://x.com/conoro), [@koltregaskes](https://x.com/koltregaskes) (score 52.6) [asking](https://x.com/koltregaskes/status/2042203406035755273): "Is it time to try OpenClaw again? I ditched it a while back."
**Coping strategies:** Import OpenClaw settings/skills into Hermes (as shown in the Hermes setup wizard), use GPT-5.4 in Hermes Agent.

### 🟡 Skill Proliferation Without Quality Signals
**Description:** Skills are proliferating faster than quality signals can keep up. [@helloitsaustin](https://x.com/helloitsaustin) (score 118.7) [warns](https://x.com/helloitsaustin/status/2042047563747225749): "be skeptical of those claiming they've got '45 agent skills that'll replace your team.' The reality is no one is coming to build your workflow for you."
**Scenario:** Developer installs 20 skills; half don't work for their specific codebase and configuration.
**Severity:** 🟡 Medium — creates noise but doesn't block.
**Prevalence:** [@rickasaurus](https://x.com/rickasaurus) (score 12.1) [predicts](https://x.com/rickasaurus/status/2041998362677846393): "this agent skills thing is point in time and will be almost completely unnecessary within a year."
**Coping strategies:** SkillsBench for evaluation, autoskills' `--dry-run`, curating skills per workflow rather than bulk installing.

### 🟢 Agent Security Delegation Model Missing
**Description:** [@chris__sev](https://x.com/chris__sev) (score 14.1) [observes](https://x.com/chris__sev/status/2042360954743116070): "The real issue is there's no delegation model at all. Every skill gets full access, no scopes, no boundaries. This is pre-OAuth all over again."
**Scenario:** Every installed skill gets the same permissions as the agent itself — no scoping.
**Severity:** 🟢 Moderate but architecturally important.
**Prevalence:** Discussed by [@chris__sev](https://x.com/chris__sev), [@KayvonJafar](https://x.com/KayvonJafar) (score 7.5): "if an agent can hallucinate a citation, it can hallucinate a wire."
**Coping strategies:** ERC-7710/7715 delegation framework ([@Clawnch_Bot](https://x.com/Clawnch_Bot)), granular Allow/Deny/Ask lists ([@geekyouup](https://x.com/geekyouup)).

## 9. Key Takeaways

1. **Skills are the new API.** Expo (+46%), Supabase, ServiceNow, MongoDB, React Native, and KuCoin all shipped agent skills packages today. This is an ecosystem-wide buildout, not isolated launches. Google Gemini's adoption of "Skills" as a UI concept confirms industry convergence.

2. **The gregisenberg tutorial is this week's defining artifact.** At 5,811 score with 1,012 bookmarks, it's the highest-engagement single piece of agent education content in the dataset. Its key insight — "skills are the actual unlock" because they load 50 tokens vs 7,000 — is reshaping how practitioners think about context management.

3. **Managed Agents face a cost wall.** The 18× pricing gap between API and subscription rates identified by [@MLStreetTalk](https://x.com/MLStreetTalk) may limit adoption to high-value enterprise workflows. This creates space for open-source alternatives like agentOS.

4. **Agent skill security is the unsolved crisis.** Skills are proliferating (autoskills installs for 50+ technologies), but there's no delegation model — every skill gets full permissions. Fangcun SkillWard is the first security scanner, but the gap between skill volume and security tooling is widening.

5. **SkillsBench proves curated skills work (+16.2pp) and self-generated don't (-1.3pp).** This is the first rigorous evidence supporting the "skills ecosystem" approach over "let the agent figure it out." It validates human curation as a moat.

6. **Pydantic's "exec not tools" approach may reshape MCP adoption.** Replacing 40 tools with 1 exec tool and getting 90% token savings challenges the assumption that more MCP tools = better agents. Watch for this pattern to spread.

7. **Agent commerce has real revenue.** bankr's $18.71M in fees, MoonPay's consecutive stablecoin onramp records, and Felix's $300K in 5 weeks prove that agent-to-agent commerce is economically viable at scale.
