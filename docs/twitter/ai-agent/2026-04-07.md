## 2. Pain Points: What Frustrates People

### Security Permissions Are Too Broad, Too Fast
**Severity: Critical | Prevalence: High**
Agents get blanket access to Gmail, Slack, GitHub, and Salesforce without the security checkpoints enterprise software normally requires. [@alex_prompter](https://x.com/alex_prompter/status/2041542927801836006): "We skipped every security checkpoint that normal enterprise software had to pass." [@JamesonCamp](https://x.com/JamesonCamp/status/2041533186195574960) quantified the risk: "12% of OpenClaw's marketplace is literal malware." Coping: Composio offers scoped auth; most users still grant blanket permissions.

### Agent Harnesses Destroy Work Without Warning
**Severity: High | Prevalence: Moderate**
[@koylanai](https://x.com/koylanai/status/2041526628468019621) lost an entire session's changes when a harness cleanup script ran `git checkout -- .` from the repo root. "The agent executed exactly what it was supposed to. The problem was that the harness had the authority to do real damage." No widely adopted guardrail exists beyond manual discipline.

### Skills Degrade in Realistic Conditions
**Severity: High | Prevalence: Emerging**
The UCSB/MIT [paper](https://x.com/dair_ai/status/2041540525539614797) showed agent skill performance degrades consistently as conditions become more realistic, with pass rates approaching no-skill baselines. Skills look great in demos with curated toolboxes but fail when agents must retrieve from large, unfiltered collections.

### Framework Fatigue Without Evals
**Severity: Moderate | Prevalence: High**
[@DerekNee](https://x.com/DerekNee/status/2041525464045658587): "We get a new agent framework every 12 hours but almost no one is building evals that keep up." Practitioners lack reliable ways to compare frameworks beyond vibes.

### Production Scaling Remains Elusive
**Severity: High | Prevalence: High**
[@ConorBronsdon](https://x.com/ConorBronsdon/status/2041527667711971721) cited the stat: 78% of enterprises have AI agent pilots, only ~14% have scaled one to production. Memory architecture, not model choice, is the bottleneck.

### Standardized Agent Security Primitives

Currently served by Composio (partial), but no industry standard exists. **Opportunity: 🔴 strong — direct.**

It needs self-optimizing harness that automates itself." No current solution auto-tunes harness parameters per model. **Opportunity: 🔴 strong — direct.**

[@DerekNee](https://x.com/DerekNee/status/2041525464045658587) articulated the gap. The Terminal-Bench 2.0 paper is a step forward but covers only skill usage. Comprehensive agent evaluation remains unmet. **Opportunity: 🟡 moderate — competitive.**

## 7. Community Sentiment

**Overall mood: Cautiously bullish, increasingly security-aware.**

The community is split along a clear axis: *builders* are enthusiastic about the expanding skill/marketplace ecosystem and the formalization of harness engineering, while *security-minded voices* are sounding alarms about the pace of capability growth outstripping safety infrastructure. The top tweet by engagement (6368.3) was a marketplace project, but several of the most substantive discussions centered on vulnerability disclosures and malware statistics.

**Divergences:**
- Crypto/AgentFi communities show much higher enthusiasm and lower skepticism than developer communities
- Reply sentiment on marketplace-related tweets often diverges from the original — promotional tweets attract critical replies about security
