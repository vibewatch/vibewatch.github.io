### 1.2 Harness Engineering 成为主导设计讨论 🡕

[@Vtrivedy10](https://x.com/Vtrivedy10) 出现在三条高互动讨论中。在 [与 @himanshustwts 的 podcast recap](https://x.com/Vtrivedy10/status/2046942634321559707)（47 个点赞、14 次收藏）中，他概括实践方式：“从模型的 capabilities/flaws 反推，并围绕它们构建 systems（harness）来完成 Tasks。”Traces 是“continual learning 和 self-improving agents 的 signal”。他还 [指出](https://x.com/Vtrivedy10/status/2046712699908944079)（19 个点赞），Google 和 frontier companies “正在强力投入 harnesses 如何/为何能让 agents 工作得更好”，并引用 [@addyosmani](https://x.com/addyosmani) 的观点：“A decent model with a great harness beats a great model with a bad harness.”

[@Marktechpost](https://x.com/Marktechpost) 通过 JiuwenClaw 的 AgentTeam capability [提出 “Coordination Engineering”](https://x.com/Marktechpost/status/2046993449262752126)（16 个点赞、33.8K 次浏览），作为 harness engineering 之后的一步：层级化编排，包含 Leader Agent、统一 team workspace 和 event-driven fault recovery。

### 1.3 CodeRabbit Agent for Slack 形成最大单产品讨论浪潮 🡕

[@harjotsgill](https://x.com/harjotsgill) [发布 CodeRabbit Agent for Slack](https://x.com/harjotsgill/status/2046955870588960969)（81 个点赞、43.5K 次浏览），[@IndianTechGuide](https://x.com/IndianTechGuide) [放大](https://x.com/IndianTechGuide/status/2046965448026272122) 到 2,104 个点赞、343 次收藏、226.8K 次浏览——当天第二高互动帖子。该产品解决 CodeRabbit 在 15,000+ teams 中观察到的三个痛点：context 和 decisions 活在 codebase 外、缺少 team-level durable knowledge base、agentic workflows 没有 trust layer。

至少六条独立引用推文复述了同一诊断。[@darshal_](https://x.com/darshal_) [写道](https://x.com/darshal_/status/2046964033199444372)（24 个点赞、14 次收藏）：“Engineering teams 通常不是因为 bad code 而挣扎。它们挣扎是因为 context gets lost。”[@base10_](https://x.com/base10_) [补充](https://x.com/base10_/status/2046963112406868454)（26 个点赞）：“AI 不只是加速了 engineering，它也把 engineering 变碎了。每个 session 都重置。每个 tool 都从零开始。你的 team 变成了 memory layer。”[@JaynitMakwana](https://x.com/JaynitMakwana) [指出](https://x.com/JaynitMakwana/status/2046961495658102889)（21 个点赞、11 次收藏）：“context switching 才是 engineering teams 真正的税。”

[@carlvellotti](https://x.com/carlvellotti) 在回复中浮现了具体 Claude Code 痛点：“1. zero memory - every session starts from scratch 2. no team context - only knows what I tell it 3. decisions die in Slack threads no one can search.”

**与前日对比：** 4 月 21 日抽象讨论了 context loss problem。4 月 22 日出现了具体产品（CodeRabbit Agent），并有六个独立账号描述同一失败模式，讨论从架构理论转向具体工具。

### 1.4 Context Engineering 从概念走向架构图 🡕

[@mdancho84](https://x.com/mdancho84) [发布](https://x.com/mdancho84/status/2046974114091389052)（39 个点赞、32 次收藏）一张三阶段演化图，区分 RAG、Agentic RAG 和 “Agentic search in context engineering”。关键区别是：context engineering 在 retrieval pipeline 中加入 file search tools、skill loading tools、database tools、web search tools、memory tools 和 shell tools，Skills YAML frontmatter 与 memory files 成为 context window 的一等组件。

![三阶段架构图，展示从固定 retrieval pipeline 的 RAG，到带 search tool 和 database 的 Agentic RAG，再到 context engineering 中带 file search、skill loading、database、web search、memory、shell tools 的 Agentic search，最终进入包含 Skills YAML frontmatter 和 memory files 的 context window](https://pbs.twimg.com/media/HGhQGs9XEAAtMvz.jpg)

[@ihtesham2005](https://x.com/ihtesham2005) [拆解 Anthropic engineering blog findings](https://x.com/ihtesham2005/status/2047057952331481423)（4 个点赞、4 次收藏），讨论 MCP context bloat：把 MCP servers 呈现为 filesystems，可将 tool definition loading 从 150,000 tokens 降到 2,000 tokens（98.7% reduction）。在 code execution environment 内先过滤数据，再进入模型，可把 Playwright snapshot 从 56KB 缩到 299 bytes，把 500-request access log 从 45KB 缩到 155 bytes。“Agent 没有变聪明。架构变干净了。”

[@_avichawla](https://x.com/_avichawla) 的 InsForge 量化结果 [继续获得互动](https://x.com/_avichawla/status/2046685172666712571)（152 个点赞、283 次收藏、38.4K 次浏览）：10.4M tokens 降到 3.7M（2.8x reduction），10 个 errors 降到 0，成本从 9.21 美元降到 2.81 美元。[@megacode_ai](https://x.com/megacode_ai) 回复：“Context engineering is becoming its own stack layer.”

**与前日对比：** 4 月 21 日给出首批量化 context engineering results（InsForge 2.8x、Claude Context 40%）。4 月 22 日增加 architecture diagrams、Anthropic 自身 engineering data（98.7% reduction）和 filesystem-as-tool-registry pattern，把 context engineering 从测量结果推进到可复现技巧。

## 2. 令人困扰的问题

### 智能体记忆需要递归式智能体管理 -- Severity: High

[@eddiegreenwood_](https://x.com/eddiegreenwood_) [描述](https://x.com/eddiegreenwood_/status/2046875748238283015)（116 个点赞、111.5K 次浏览），自己为 agent memory 构建了 filing system，结果又需要另一个 agent 来管理它。“agent on top of agent. starting to feel like im chasing the dragon.”[@OmoKadupe05](https://x.com/OmoKadupe05) 指出更深层问题：“什么时候添加 agents 来管理 memory，只是在重建你试图解决的复杂度？”[@_orcaman](https://x.com/_orcaman) [补充](https://x.com/_orcaman/status/2046879308161925565)（6 个点赞）：“agent memory sucks……总把 irrelevant background 混进 fresh conversations。”

Prevalence: High -- 多个独立账号描述同样的 recursive management problem。没有当前 framework 能在没有 human oversight 的情况下提供可扩展 memory curation。

### 基础设施定价压缩威胁智能体创业公司 -- Severity: High

[@aakashgupta](https://x.com/aakashgupta) [延续](https://x.com/aakashgupta/status/2046694977225506975)（27 个点赞、32 次收藏）4 月 21 日的结构性论点：Anthropic 每小时 0.08 美元、Microsoft Foundry hypervisor-level isolation、Google Gemini Enterprise Agent Platform 都在同一周收敛。“kill zone 中的公司可能只剩 12-18 个月 runway。”[@gagansaluja08](https://x.com/gagansaluja08) 确认：“infra is commodifying fast。真正问题是哪一个平台会拉走 developer surface。”

Prevalence: 对数据集中识别出的 30-50 家 agent infrastructure startups 来说是结构性问题。三方 hyperscaler convergence 让它比 4 月 21 日更尖锐。

### 工程工具之间的上下文丢失被 AI Agents 放大 -- Severity: Medium

4 月 22 日，六个独立账号描述了同一失败模式。[@carlvellotti](https://x.com/carlvellotti)：“zero memory - every session starts from scratch……decisions die in Slack threads.”[@JaynitMakwana](https://x.com/JaynitMakwana)：“context switching is the actual tax on engineering teams. slack to terminal to github, back to slack.”[@base10_](https://x.com/base10_)：“AI didn't just speed up engineering, it fragmented it.”

Prevalence: 在使用 AI coding agents 的工程团队中很高。CodeRabbit Agent 是第一个明确针对这一缺口的已发布产品。

### Harness Engineering 缺少职业身份 -- Severity: Low

[@TheAhmadOsman](https://x.com/TheAhmadOsman) 宣称 “Harness Engineering is the new Software Engineering”，但回复抵触这种 framing。[@guigotgit](https://x.com/guigotgit)：“we have yet to find a better name for it.”[@rugbist_](https://x.com/rugbist_)：“same job but with more steps and probably lower pay.”[@ethankongee](https://x.com/ethankongee) 提出评估缺口：“我找到的大多数文章都太抽象……没有扎实 benchmarks，很难知道一个 harness 是否真的好。”

Prevalence: Emerging -- 这门学科已存在，但命名和标准化评估标准都还没有共识。

## 3. 人们期望的功能

### 不需要递归管理的自我整理智能体记忆

[@eddiegreenwood_](https://x.com/eddiegreenwood_/status/2046875748238283015)（116 个点赞、111.5K 次浏览）搭了自己的 filing system，却仍需要一个 agent 管理它。核心缺口是：没有 framework 能自动区分什么值得记住、什么是噪声，同时又不需要新增一个 agent 层。[@eddiegreenwood_](https://x.com/eddiegreenwood_) 在回复中总结：“memory is easy. curation is the job.”

Opportunity: High -- 一个带 built-in relevance scoring 和 automatic pruning 的 memory system，如果能在单一 agent loop 内运作，而不是独立 management layer，将解决“chasing the dragon”模式。

### 标准化 Harness Engineering Benchmarks

[@ethankongee](https://x.com/ethankongee) [描述缺口](https://x.com/TheAhmadOsman/status/2046721205344850406)：“我正在学习如何构建 agent harnesses，但找到的大多数文章都太抽象，没法帮助我判断 harness 是否有效。没有扎实 benchmarks，很难知道一个 harness 是否真的好。”[@SentientAGI](https://x.com/SentientAGI) 的 Arena Cohort 0 results 是一个开端，但还没有公开 benchmark 能把 harness quality 和 model quality 分离出来。

Opportunity: High -- 一个能把 agent performance 分解为 model contribution 与 harness contribution 的 benchmark，将重塑团队如何评估和投资 agent infrastructure。

### Voice Agent Interruption and State Management

[@JamesClawn](https://x.com/JamesClawn) [指出](https://x.com/Cloudflare/status/2046816309963378840) 一个具体 failure mode：“当 interruption 停止 audio 但没有停止 pending tool call 时，voice agents 会破坏 trust，因为 memory 和 turn state 仍然需要 hard revoke，agent 才不会继续行动。”[@somi_ai](https://x.com/somi_ai) 确认：“大多数 voice stacks 在用户想 barge in 时，仍会把用户截断在半个 syllable。”

Opportunity: Medium -- 在 tool-call level 而不只是 audio level 处理 interruption 的 voice agent frameworks，将解决 production voice deployments 的结构性 trust gap。

### Notable project details

**Hyperframes by HeyGen** 是一个创造品类的发布：第一个从一开始就为 AI agents 设计的视频框架。此前每个视频工具都默认有一个拿鼠标的人。Hyperframes 使用 HTML 作为 composition format，用 data attributes 定义 timing，用 elements 定义 layers。它为 Claude Code、Cursor、Gemini CLI 和 Codex 提供 agent skills，编码了 framework-specific patterns。Apache 2.0 license，发布时 npm downloads 为 24K/month。

![HeyGen Hyperframes README，展示 HTML-to-video rendering framework with first-class AI agent support，npm badges 显示 v0.4.12、24K/month downloads、Apache 2.0 license](https://pbs.twimg.com/media/HGf_jLRaUAAPioC.png)

**OpenHarness** 填补了 closed-source Claude Code 与极简 custom harnesses 之间的空白。架构定义为 Harness = Tools + Knowledge + Observation + Action + Permissions。它包含 43+ tools、与 Anthropic 格式兼容的 skills、与 claude-code plugins 兼容的 plugin、multi-level permissions（default/auto/plan modes），并附带 ohmo——一个 personal agent，可在 Feishu、Slack、Telegram 或 Discord 中使用既有 Claude Code 或 Codex subscriptions 工作。

![OpenHarness 架构，展示 Harness equation：Tools（43+，含 bash、read、write、search）+ Knowledge（skills、CLAUDE.md、memory）+ Observation（git diff、error logs、file state）+ Action（CLI commands、API calls、file edits）+ Permissions（sandboxing、approval、trust）](https://pbs.twimg.com/media/HGgnQDZbUAAjzgG.jpg)

## 6. 新动态与亮点

### Devin 反转对多智能体系统的立场

[@cognition](https://x.com/cognition)（Devin 背后的公司）[宣布](https://x.com/cognition/status/2047054526583050647)（44 个点赞、38 次收藏、7.2K 次浏览）已经实现 multi-agent flows，反转其 CPO 之前的公开立场。[@walden_yan](https://x.com/walden_yan) [限定了这种变化](https://x.com/walden_yan/status/2047054554433462360)（14 个点赞）：“many sexy ideas are still impractical, but we've found some setups that actually work.” 这种选择性背书——multi-agent 用于 specific flows，而不是通用架构——标志着最大融资 coding agent 公司的一次重要转向。

### Open Models 在 Arena Competition 中以每美元性能接近 Frontier

[@SentientAGI](https://x.com/SentientAGI) [发布](https://x.com/SentientAGI/status/2046995270039482534)（72 个点赞、3.2K 次浏览）Arena Cohort 0 competition results。Top team 用 MiniMax M2.5 以 1.78 美元/次达到 69.91% accuracy；而 Claude Opus 4.5 以 55.44 美元/次达到 80.89%。最终 composite scores 几乎相同（188.11 vs 187.44），说明 harness engineering、prompt density 和 skills 能以极低成本弥补模型能力差距。

### Qwen3.6 27B 在 Coding Agent Benchmarks 上大幅提升

[@leftcurvedev_](https://x.com/leftcurvedev_) [报告](https://x.com/leftcurvedev_/status/2046948277145235868)（16 个点赞），Qwen3.6 27B 相比 Qwen3.5 27B 在多个 agent benchmarks 上有显著提升：Terminal-Bench 2.0 +42.55%、SkillsBench +77.21%、QwenWebBench +39.23%、NL2Repo +32.60%、Claw-Eval +12.60%。该版本还引入 retained reasoning context from historical messages，降低 iterative agentic workflows 的开销。

### ERC-8226 提出 AI Agent 金融操作监管框架

[@Brickken](https://x.com/Brickken) [介绍](https://x.com/Brickken/status/2046955624949317899)（35 个点赞）ERC-8226，一个 compliant agent mandates 框架，允许 verified investors 把有限 on-chain authority 委派给 AI agents，同时保留 KYC、AML、eligibility checks 和 issuer control。这是第一个针对金融市场中 regulated AI agent operations 的具体 Ethereum standards proposal。

### MCP Context Bloat 获得 98.7% 降低技巧

[@ihtesham2005](https://x.com/ihtesham2005) [综合](https://x.com/ihtesham2005/status/2047057952331481423) Anthropic engineering blog findings：把 MCP servers 作为 filesystems 呈现，而不是一次性加载所有 tool definitions，可将 context 从 150,000 tokens 降到 2,000 tokens。Cloudflare 独立确认同一模式，称之为 “Code Mode”——让 agents 编写代码来调用 tools，而不是直接调用 tools。

## 7. 机会在哪里

`[+++]` **智能体记忆整理系统** -- Recursive management problem 是当天最高浏览量挫败信号（111.5K views）。每个 agent framework 都提供 memory storage，但没有一个提供 intelligent curation——决定什么值得记住、何时遗忘，以及如何防止 irrelevant context 污染 fresh sessions。一个带 built-in relevance scoring 和 automatic pruning、且在单一 agent loop 内运行的 memory system，将解决“chasing the dragon”模式。Sources: [@eddiegreenwood_](https://x.com/eddiegreenwood_/status/2046875748238283015), [@_orcaman](https://x.com/_orcaman/status/2046879308161925565), [@WalrusProtocol](https://x.com/WalrusProtocol/status/2047010404040405248).

`[+++]` **Context engineering tooling and services** -- 三个独立数据点确认 context waste 巨大：InsForge 的 2.8x token reduction、Anthropic 的 98.7% tool-definition reduction、Cloudflare 的 Code Mode pattern。构建 context optimization layers 的团队——semantic retrieval、filesystem-as-registry patterns、incremental indexing、context compression——同时解决成本和可靠性。市场仍早且碎片化。Sources: [@_avichawla](https://x.com/_avichawla/status/2046685172666712571), [@ihtesham2005](https://x.com/ihtesham2005/status/2047057952331481423), [@mdancho84](https://x.com/mdancho84/status/2046974114091389052).

`[++]` **Harness benchmarking and evaluation** -- Arena Cohort 0 results 显示 harness quality 在最终 performance scores 上可与 model choice 相提并论，但还没有公开 benchmark 能 isolating harness contribution from model contribution。一个标准化 harness evaluation framework 将帮助团队决定在 infrastructure 还是 model spending 上投入。[@ethankongee](https://x.com/ethankongee) 明确要求“不同 tasks 和 industries 的 benchmarks”。Sources: [@SentientAGI](https://x.com/SentientAGI/status/2046995270039482534), [@ethankongee](https://x.com/TheAhmadOsman/status/2046721205344850406).

`[++]` **工程工作流的团队上下文持久化** -- 六个独立账号描述了同一失败：AI coding sessions 重置 context、decisions 死在 Slack threads 中、teams 每个 sprint 反复解释同样 context。CodeRabbit Agent for Slack 是首个针对该问题的已发布产品，但问题延伸到 Slack 之外的所有 collaboration surfaces。Sources: [@harjotsgill](https://x.com/harjotsgill/status/2046955870588960969), [@carlvellotti](https://x.com/carlvellotti), [@base10_](https://x.com/base10_/status/2046963112406868454).

`[+]` **Agent-native video and media creation** -- HeyGen 的 Hyperframes 是第一个让 agents 创建视频的开源框架。更广泛模式是：所有 media creation tools 都是为拿鼠标的人构建的，而 agents 需要 native interfaces。Audio（ElevenLabs skills）、video（Hyperframes）和 image tools 如果设计成 agent primitives，将在增长中的 UGC automation wave（Higgsfield、Hermes Agent）中有即时需求。Sources: [@sentient_agency](https://x.com/sentient_agency/status/2046885537878065511), [@ElevenLabsDevs](https://x.com/ElevenLabsDevs/status/2047025917034209382).

`[+]` **Tool-call level 的 voice agent state management** -- 当前 voice agent frameworks 在 audio level 处理 interruption，但不在 tool-call level 处理；当被打断的 agent 继续执行 pending actions 时，会破坏 trust。Cloudflare 和 LiveKit 正在投入这一方向，但 customer support 和 telephony 等 production voice deployments 仍有缺口。Sources: [@JamesClawn](https://x.com/JamesClawn), [@Cloudflare](https://x.com/Cloudflare/status/2046816309963378840), [@livekit](https://x.com/livekit/status/2047048713638453591).

## 8. 要点总结

1. 三家 hyperscalers 在同一个 24 小时窗口内发布企业智能体平台。Microsoft Foundry Agents（persistent microVMs、Entra identity、1,000+ tools）、Google Gemini Enterprise Agent Platform（与 Atlassian、Oracle、ServiceNow 合作的 agent marketplace）以及 Anthropic 的 0.08 美元/小时 runtime，都瞄准同一层。4 月 21 日识别出的基础设施定价压缩，现在已成为三方竞赛。Sources: [@satyanadella](https://x.com/satyanadella/status/2047033636923568440), [@georgeorch](https://x.com/georgeorch/status/2047002831387853017), [@aakashgupta](https://x.com/aakashgupta/status/2046694977225506975).

2. Harness engineering 以量化证据主导设计讨论。Arena Cohort 0 显示，open-source MiniMax M2.5 在合适 harness 下，以 Claude Opus 4.5 成本的 1/30 获得接近 frontier 的 performance。JiuwenClaw 提出 “Coordination Engineering” 作为下一层抽象。新共识是：按美元计，harness quality 比 model selection 更重要。Sources: [@SentientAGI](https://x.com/SentientAGI/status/2046995270039482534), [@Vtrivedy10](https://x.com/Vtrivedy10/status/2046942634321559707), [@Marktechpost](https://x.com/Marktechpost/status/2046993449262752126).

3. CodeRabbit Agent for Slack 产生当天最大单产品讨论浪潮（amplified post 上 2,104 个点赞、226.8K 次浏览）。六个独立账号描述同一痛点：AI coding agents 把 engineering context 碎片化到多个工具中，teams 变成 memory layer。Sources: [@IndianTechGuide](https://x.com/IndianTechGuide/status/2046965448026272122), [@harjotsgill](https://x.com/harjotsgill/status/2046955870588960969), [@base10_](https://x.com/base10_/status/2046963112406868454).

4. Context engineering 从测量结果推进到可复现架构模式。Anthropic 自身 engineering data 显示，通过把 MCP servers 呈现为 filesystems，可降低 98.7% context。InsForge 以 2.8x token reduction 继续获得互动。[@mdancho84](https://x.com/mdancho84) 的架构图把从 RAG 到 agentic context engineering 的演进编码成三阶段 pipeline。Sources: [@ihtesham2005](https://x.com/ihtesham2005/status/2047057952331481423), [@_avichawla](https://x.com/_avichawla/status/2046685172666712571), [@mdancho84](https://x.com/mdancho84/status/2046974114091389052).

5. 智能体记忆成为当天最高挫败的未解问题。[@eddiegreenwood_](https://x.com/eddiegreenwood_) 用 111.5K 次浏览描述 recursive management pattern——构建 agents 来管理 agent memory。当前没有 framework 能在不添加管理层的情况下提供可扩展 memory curation。Source: [@eddiegreenwood_](https://x.com/eddiegreenwood_/status/2046875748238283015).

6. Devin 反转了对 multi-agent systems 的公开立场，在 10 个月前反对之后实现了 “specific flows”。限定词很重要：multi-agent 适用于 targeted use cases，而不是通用架构。这与当天更广泛的 selective、evidence-based adoption 模式一致。Sources: [@cognition](https://x.com/cognition/status/2047054526583050647), [@walden_yan](https://x.com/walden_yan/status/2047054554433462360).

7. 官方 agent skills repositories 来自 Google Cloud（13 products）、ElevenLabs（voice isolation）和 .NET（三种 authoring modes）。Crypto Skill Hub 记录了 97 个 MCP servers 上的 1,185 个 skills。Skills 生态正从 community-driven 转向 vendor-curated，`npx skills add` 成为标准安装模式。Sources: [@rseroter](https://x.com/rseroter/status/2047062761738494073), [@ElevenLabsDevs](https://x.com/ElevenLabsDevs/status/2047025917034209382), [@diegoxyz](https://x.com/diegoxyz/status/2046885322685067390).

8. HeyGen 开源 Hyperframes，第一个为 AI agents 构建的视频渲染框架。HTML in、MP4 out，并为 Claude Code、Cursor、Gemini CLI 和 Codex 提供 skills。与此同时，Higgsfield Marketing Studio（Hermes Agent-powered UGC creation）产生多个高互动帖子，声称能以 100 美元/月替代 500 美元/视频的 agency work。Source: [@sentient_agency](https://x.com/sentient_agency/status/2046885537878065511).
