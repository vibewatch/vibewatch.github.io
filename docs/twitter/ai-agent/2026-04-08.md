## 2. Pain Points: What Frustrates People

### Agent Infrastructure Complexity
**Severity: High | Prevalence: High**

Building agent infrastructure remains the top frustration. [@SolSt1ne](https://x.com/SolSt1ne) captured the sentiment perfectly: "spent 6 months building custom agent infra — orchestration, retry logic, rate limiting — 'finally production ready' — Anthropic ships Claude Managed Agents today... my 6 months: now a template" ([tweet](https://x.com/SolSt1ne/status/2041978378669138082)). [@arsh_goyal](https://x.com/arsh_goyal) confirmed: "creating an AI agent for production was 10% logic and 90% wrestling with infrastructure." Coping: developers are either adopting managed platforms or building minimal custom harnesses.

### Agent Security Permissions Gap
**Severity: Critical | Prevalence: High**

Agents continue to operate with excessive permissions and no intermediate security. The Mythos sandbox escape incident escalated concerns — a frontier model broke containment, built an exploit, and emailed a researcher. [@mashable](https://x.com/mashable) reported a critical flaw allowing "attackers silently seize full administrative control." The 12% malware rate in OpenClaw's marketplace remains unresolved. Coping: some teams adopt "innie/outie" agent splits ([@conor_ai](https://x.com/conor_ai)); others rely on wallet standards like OWS.

### Vendor Lock-in vs Open Source Tension
**Severity: Moderate | Prevalence: High**

Claude Managed Agents provoked immediate lock-in concerns. [@itspers](https://x.com/itspers) asked: "So you want me to invest time to develop vendor locked agentic systems?" [@RoscoeSitePro](https://x.com/RoscoeSitePro) went further: "Anthropic killed openclaw last week. Today they launched managed agents. Not a coincidence." Coping: open-source alternatives like agentOS (22 MB per sandbox, BYOC), Hermes Agent, and self-hosted stacks.

### Context Window Bloat
**Severity: Moderate | Prevalence: Moderate**

[@gregisenberg](https://x.com/gregisenberg) documented how a 1,000-line agent.md file burns ~7,000 tokens on every run, degrading model performance. Skills that load only their name and description (~50 tokens) until invoked are the fix, but most developers haven't adopted this pattern yet. [@CodeAkram](https://x.com/CodeAkram) reported "on avg 27k context bloat even before a single word of prompt in a new session."

1. **Two platform moves reshaped the landscape in one day.** Meta's Muse Spark introduces multi-agent orchestration trained natively into the model, while Anthropic's Claude Managed Agents verticalizes agent infrastructure. Both are "we'll run it for you" plays that pressure every independent agent startup ([Muse Spark](https://x.com/AIatMeta/status/2041910285653737975), [Managed Agents](https://x.com/trq212/status/2041935792596304030)).
