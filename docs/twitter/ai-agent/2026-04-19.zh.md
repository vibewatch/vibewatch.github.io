### 1.2 自我改进智能体协议正式化 🡕

三条独立讨论汇聚到了带有安全护栏的自我改进智能体话题上。[@omarsar0](https://x.com/omarsar0) 分享了一篇[关于 Autogenesis 的论文](https://x.com/omarsar0/status/2045956901750399374)（92赞，125收藏，10,300浏览）——当日最高收藏数——称其为"一个用于提出、评估和提交改进的框架，具有可审计的血统追踪和回滚机制。"

![Autogenesis 架构图，展示自我改进循环：反思、选择、改进、评估、提交/回滚，GAIA得分89.04](https://pbs.twimg.com/media/HGSyU8QXQAAUNdJ.jpg)

该图展示了版本化的智能体组件（提示词、工具、记忆）通过上下文管理器流向提交或回滚门。自我改进循环依次运行：反思（读取运行轨迹）> 选择（选定一个编辑目标）> 改进（起草候选方案）> 评估（运行测试或判断）> 提交/回滚（保留或撤销，保存证据）。报告显示在 GAIA Test 上平均 Pass@1 为89.04。[@Shurtcurt](https://x.com/Shurtcurt) 评论道："底部'带着历史重复，而非失忆'这条注释承担了大量工作。大多数自我改进智能体设计之所以失败，是因为每个改进周期都从零开始。"[@damoosmann](https://x.com/damoosmann) 反驳称"这些论文的评分依据是研究生选的代理指标；循环针对该指标优化直到饱和，而增益无法迁移。"

[@jiqizhixin](https://x.com/jiqizhixin) 报道了[清华大学的 AutoSOTA](https://x.com/jiqizhixin/status/2045915580562296921)（9赞，8收藏）：一个由八个专业智能体组成的多智能体系统，可阅读研究论文并构建改进版本。它在 LLM、计算机视觉和时间序列领域发现了105个新的 SOTA 模型，平均每篇论文耗时五小时。

![AutoSOTA 架构图，展示由八个专业智能体组成的四阶段自动化研究流程](https://pbs.twimg.com/media/HGGMjbMbAAAkVtH.jpg)

**讨论要点：** 4月19日，自我改进智能体从概念阶段进入了协议层面。Autogenesis 为智能体自我修改引入了类 git 语义（提交、回滚、血统追踪）。AutoSOTA 展示了该概念可产出可量化的研究成果。代理评估与实际迁移效果之间的矛盾仍未解决。

**与前日对比：** 4月18日介绍了 GenericAgent（自我进化，约3,000行代码）作为已发布的框架。4月19日新增了学术协议层（Autogenesis）和首个自动化研究产出证据（AutoSOTA 的105个新 SOTA 模型）。讨论从"会学习的智能体"成熟为"能安全地带审计轨迹重写自身的智能体"。

### 1.7 规范驱动开发进入智能体工作流 🡕

## 3. 人们期望的功能

### 智能体原生操作系统

[@DanGrover](https://x.com/DanGrover) 提问：["如果把智能体及其上下文作为一等构造来设计，操作系统会是什么样子？"](https://x.com/DanGrover/status/2045963531628220742) 他描述了当前的摩擦："我花大量时间把应用和服务中的内容导入到某个智能体中。这感觉就像在垃圾回收时代还在手动管理内存分配。"目前尚无人构建这样的系统。

机会：一个将智能体上下文作为一等对象的操作系统级抽象——具备生命周期管理、跨智能体权限和共享内存池——将消除每个智能体构建者目前都在承担的集成税。

### 自动技能发现与匹配

[@SteveSolun](https://x.com/SteveSolun) 构建了 [ctx](https://x.com/SteveSolun/status/2045966266519978170)，它"监控你正在开发的内容，遍历一个包含1,450+技能和427个智能体的知识图谱，实时推荐合适的技能。"1,450个技能已经存在但发现它们需要专用工具，这说明市场问题现在是一个搜索问题。

机会：能将开发上下文与相关技能匹配的技能推荐引擎，将减少快速增长但索引不完善的生态系统中的使用摩擦。

### 标准化智能体身份与委托

[@GustavoValverde](https://x.com/GustavoValverde) 发布了[两份互联网草案](https://x.com/GustavoValverde/status/2045654205969453179)：PACT（Private Agent Consent and Trust Profile，基于 OAuth 2.1 的隐私保护智能体委托）和 VEIL（Verified Ephemeral Identity Layer）。参与度较低（0赞，1收藏），但结构性重要性很高。

机会：没有标准化的智能体身份和委托机制，每次智能体间交互都需要定制化的信任协商。基于 OAuth 的配置文件将实现跨平台的可移植智能体认证。

## 6. 新动态与亮点

### Hermes Agent 以创纪录速度达到10万Star

[Hermes Agent](https://github.com/NousResearch/hermes-agent) 在53天内达到100,000 GitHub Star，由 [@Teknium](https://x.com/Teknium/status/2045739848913600640) 确认，并可在 [GitHub Trending 截图](https://x.com/RoundtableSpace/status/2045906520672461309) 中看到101,400 Star，本周新增42,612。该仓库自我定位为"自我改进的AI智能体"，具有闭环学习循环：智能体策展的记忆、自主技能创建、使用中的技能自我改进和 FTS5 会话搜索。通过单一网关支持 Telegram、Discord、Slack、WhatsApp、Signal 和 CLI。

### GitHub Trending 前五全部与智能体相关

数据集中首次出现[前五名热门仓库全部为AI智能体项目](https://x.com/RoundtableSpace/status/2045906520672461309)。andrej-karpathy-skills（一个改进 Claude Code 行为的单独 CLAUDE.md 文件）一周内获得44,465 Star，证明即使是最简单的技能产物，只要解决了普遍问题，就能实现大规模传播。

### Autogenesis 协议引入智能体自我改进的 Git 机制

[@omarsar0](https://x.com/omarsar0) 展示了 [Autogenesis](https://x.com/omarsar0/status/2045956901750399374)（125收藏），一个智能体可以提出、评估和提交改进的协议，具有可审计的血统追踪和回滚机制。五步循环（反思 > 选择 > 改进 > 评估 > 提交/回滚）配合"带着历史重复，而非失忆"的原则，代表了首个安全智能体自我修改的正式协议。

### AutoSOTA 自动发现105个新 SOTA 模型

[@jiqizhixin](https://x.com/jiqizhixin) 报道了[清华大学的 AutoSOTA](https://x.com/jiqizhixin/status/2045915580562296921)：一个八智能体系统，可阅读研究论文并构建改进版本，在 LLM、计算机视觉和时间序列领域发现了105个新的最先进模型，平均每篇论文耗时五小时。

### AgentShield 发布首个具体威胁统计数据

[AgentShield](https://x.com/tom_doerr/status/2045815402735845874)（npm v1.4.0，月下载量4,500次）发布了威胁数据：2026年1月某主要技能市场12%为恶意内容，一个 CVSS 8.8 的 CVE 暴露了17,500+个实例，Moltbook 泄露事件导致150万个 API token 被盗。这是首批针对智能体生态系统的公开量化威胁统计数据。

### 前沿模型在智能体安全基准测试中失败

[@holoengineAI](https://x.com/holoengineAI) 对单独运行的 GPT-5.4、Claude Sonnet 4.6 和 Gemini 2.5 Pro 进行了金融决策任务[基准测试](https://x.com/holoengineAI/status/2045961439731720584)。三者均返回了错误的 ALLOW 判定。只有 Holo 1.1（配备对抗性验证循环）正确触发了升级。

## 8. 要点总结

1. Hermes Agent 在53天内达到10万 GitHub Star，前五名热门仓库全部与智能体相关。AI智能体生态系统现在是 GitHub 上的主导力量。来源：[@Teknium](https://x.com/Teknium/status/2045739848913600640)、[@RoundtableSpace](https://x.com/RoundtableSpace/status/2045906520672461309)。

3. 智能体记忆基础设施达到临界规模，Cognee 登上 Trending 第一（15K Star），claude-mem 达到63K Star，OpenAI 工程师发表记忆架构演讲。记忆现在是智能体平台之间的首要差异化因素。来源：[@tom_doerr](https://x.com/tom_doerr/status/2045701466472493534)、[@Av1dlive](https://x.com/Av1dlive/status/2045909034184954310)。

4. Claude Code 速率限制正在推动开发者转向使用开源模型（GLM 5.1、Gemma 4）的免费替代方案，具体用户不满已在一个20条回复的讨论串中记录。来源：[@Finstor85](https://x.com/Finstor85/status/2045758628196647387)、[@JulianGoldieSEO](https://x.com/JulianGoldieSEO/status/2045963288748724686)。

6. 通用智能体框架（KohakuTerrarium、AgentScope）通过配置驱动的智能体创建挑战了从零构建的文化，而规范驱动开发（SDD Skills）将代码视为可丢弃的编译目标。来源：[@KBlueleaf](https://x.com/KBlueleaf/status/2045838664920707315)、[@sivalabs](https://x.com/sivalabs/status/2045732980162011216)。
