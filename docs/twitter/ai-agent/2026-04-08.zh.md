# Twitter AI 智能体 - 2026-04-08

## 1. 人们在讨论什么

### 1.1 Meta 发布 Muse Spark，内置多智能体编排 🡕

Meta Superintelligence Labs 发布了 [Muse Spark](https://x.com/AIatMeta/status/2041910285653737975)，这是一个原生多模态推理模型，具备工具使用、视觉思维链和多智能体编排能力。公告获得 152K+ 浏览量，并主导了当天讨论。[@alexandr_wang](https://x.com/alexandr_wang)（Scale AI CEO）[提供了技术分析](https://x.com/alexandr_wang/status/2041909380208324687)，指出 Muse Spark 展示了“pretraining、RL 和 test-time reasoning 之间可预测的 scaling”，并引入“thought compression”——用显著更少 token 解决问题的能力。

![Muse Spark benchmark 对比表](https://pbs.twimg.com/media/HFZT0TMa8Ac6iLW.png)

[@jhyuxm 分享的 benchmark 表](https://x.com/jhyuxm/status/2041913529033486468)显示，Muse Spark 在 CharXiv Reasoning（86.4 vs 65.3）、ScreenSpot Pro（84.1 vs 83.1）和 HealthBench Hard（42.8 vs 14.8）上领先 Opus 4.6，但在 ARC AGI 2（42.5 vs 63.3）和 GPQA Diamond（89.5 vs 92.7）上落后。该模型的“Contemplating mode”会编排多个智能体并行推理；[@omarsar0](https://x.com/omarsar0) [指出](https://x.com/omarsar0/status/2041919769536770247)，这些智能体“在不显著增加延迟的情况下实现并行化”，并分享了一张图：16 个并行智能体在 Humanity's Last Exam 上达到约 59%，单智能体约为 55%。

![Humanity's Last Exam 多智能体 scaling 图](https://pbs.twimg.com/media/HFZZM-ZXgAA3X9t.png)

质疑很快出现。一条[回复](https://x.com/ns123abc/status/2041920450536612282)指出 Meta “花了 $14 billion，挖走 Alexandr Wang，从零重建整个 stack。第一个模型落在第四名。” [@omarsar0](https://x.com/omarsar0) 标记出“一些缺口，尤其是要支持 long-horizon agentic systems 和 coding workflows。” [@TrendSpider 观察到](https://x.com/TrendSpider/status/2041919844149244323)，Meta 股价因这则消息大幅反弹。

### 1.2 Anthropic 发布 Claude Managed Agents，威胁编排创业公司 🡕

Anthropic 发布了公开 beta 的 [Claude Managed Agents](https://x.com/claudeai/status/2041927687460024721)，社区反应立刻爆发。[@trq212](https://x.com/trq212) 称其为[第一个在简洁性和复杂性之间取得正确平衡的“云端 agent API”](https://x.com/trq212/status/2041935792596304030)，并详细说明了带自定义 packages、credential vaults、file system memory 和 GitHub integration 的 environments。这条推文获得 921 个点赞和 402 个收藏。

[@aakashgupta](https://x.com/aakashgupta) 宣称 Anthropic [“刚刚批量淘汰了所有 agent orchestration startup”](https://x.com/aakashgupta/status/2041940149328834748)，并指向一个 production fleet dashboard：8 个 agents 正在运行，247 个 tasks 已完成，并接入 MCP-connected HubSpot integrations。“Manus 花了六个月做五次 harness 重写。LangChain 花了一年做四套架构。Anthropic 直接发布了 managed version，让你根本不用再自己构建。” [@shiri_shh](https://x.com/shiri_shh) 更直接：[“整个 AI agent startup sector 今天被做熟了。”](https://x.com/shiri_shh/status/2041966677634711768)

架构争论也随之升温。[@charlespacker](https://x.com/charlespacker) [分享了 Anthropic 工程博客摘录](https://x.com/charlespacker/status/2041941666802889086)，描述“pet vs cattle”的 container 问题——当智能体组件共享一个 container 时，“harness 中的 bug、event stream 中的 packet drop，或者 container 离线，看起来都会呈现同一种 failure mode。” [@alexgshaw](https://x.com/alexgshaw) [发问](https://x.com/alexgshaw/status/2041946042040775167)：“谁在构建‘Agent Sandbox Protocol’？”

![Anthropic 工程博客中的 container 架构](https://pbs.twimg.com/media/HFZuaDIa8AAJwf-.jpg)

[Notion 宣布](https://x.com/NotionHQ/status/2041982872698155398)接入 Claude agents，把 Notion 定位为编排层，而 Anthropic 负责运行模型和 harness。社区成员马上追问是否会支持其他模型。

### 1.3 Skills 生态爆发（裂缝也显现）🡕

Agent skills 成为扩展智能体能力的主导范式。[@gregisenberg](https://x.com/gregisenberg) 发布了一篇[详细拆解](https://x.com/gregisenberg/status/2042010418454184325)（398 个收藏），认为 agent.md files “大多没必要”——一个 1000 行文件每次运行会烧掉 7000 token，而 skill 在激活前只加载 name 和 description（约 50 token）。他的框架是：先手动跑一遍 workflow，让智能体写 skill，再一起递归修复失败。

![GitHub 上的 AI Agent Skills library](https://pbs.twimg.com/media/HFWMT4GWsAAZdkM.png)

工具层正在快速成熟。[@tom_doerr](https://x.com/tom_doerr) 分享了 [AI Agent Skills](https://x.com/tom_doerr/status/2041692281493667984)（991 stars、64 skills、11K downloads）和 [Skill Forge](https://x.com/tom_doerr/status/2042003146885525856)，后者是一条可以同时审计并发布 skills 到五个平台的 pipeline。Skill Forge 的 README 揭示了一个冷静数据：生态中 88K+ 已发布 skills 里，社区审计估计约 26% 存在安全漏洞。[@FOUNDATIONdvcs](https://x.com/FOUNDATIONdvcs) [证实](https://x.com/FOUNDATIONdvcs/status/2041927970684600331)：“AI agent marketplace：12% 是 malware。”

![Skill Forge 审计和发布 pipeline](https://pbs.twimg.com/media/HFanCoGWkAA_rEr.png)

多个 vendors 宣布支持 skills：[MongoDB for Cursor](https://x.com/MongoDB/status/2041878733108195335)、带 MCP server 的 [Coinbase Developer Platform](https://x.com/AlphaCartell/status/2041952474269995028)、用于 Base 和 Farcaster 上 onchain mini apps 的 [Lazer CLI](https://x.com/GSkrovina/status/2041935188536549403)，以及在一次更新里交付 skills + cloud + MCP 的 [Enter Pro](https://x.com/EnterProAI/status/2041848538473361473)。

### 1.4 Harness Engineering 和 Context Engineering 形式化 🡒

两个相关学科正在以正式名称结晶。[@latentspacepod](https://x.com/latentspacepod) 发布了 [“Extreme Harness Engineering”](https://x.com/latentspacepod/status/2041567210871959672)，介绍 OpenAI 的 Symphony——一个由 Codex agents 生成 100 万行代码、每天 10 亿 token 的“ghost library”，做到“0% human code, 0% human review pre merge”。访谈揭示了 harness engineering 如何在 OpenAI 内部从 GPT-5 演进到 GPT-5.4。

[@IntuitMachine](https://x.com/IntuitMachine) 分享了一套 [L1-L6 成熟度模型](https://x.com/IntuitMachine/status/2041503368695185720)，将从 prompt engineering（L1）到 context engineering（L2）、harness engineering（L3-L4），再到 self-improving agents（L6）的进展映射出来，并把“Quality/Vitality Engineering”识别为 L6 阶段尚未命名的行业缺口。

![AI engineering maturity model L1-L6](https://pbs.twimg.com/media/HFTgey-agAAKLvz.jpg)

[@helloiamleonie](https://x.com/helloiamleonie)（Elastic）在 [AI Engineer Europe 举办了一个 workshop](https://x.com/helloiamleonie/status/2041859479654879524)，主题是“Agentic Search for Context Engineering”。[@DSPyOSS](https://x.com/DSPyOSS) [声称](https://x.com/DSPyOSS/status/2041635988955377669)自己预判了每一波：“2024: agentic workflows! us: DSPy. 2025: context engineering! us: DSPy. 2026: harness engineering!! us: DSPy.”

![Agentic Search for Context Engineering workshop slide](https://pbs.twimg.com/media/HFYbBMGa8AAMbNF.jpg)

### 1.5 智能体基础设施：S3、沙箱和 OS 问题 🡕

[@skeptrune](https://x.com/skeptrune) [认为](https://x.com/skeptrune/status/2041657917708103915)，Amazon S3 Files 对智能体具有转型意义：“你不再需要启动 sandbox vm，才能让智能体访问 POSIX tools。现在你可以把任意规模的 compute 指向 s3，在同一个 filesystem 上运行大规模并行 agent swarms。” 这条推文获得 632 个收藏——当天最高 bookmark rate。讨论也降低了预期，指出你仍然需要 sandbox 来 mount filesystem。

[@NathanFlurry](https://x.com/NathanFlurry) [描述](https://x.com/NathanFlurry/status/2041985364454469919)了正在构建的开源 sandbox infrastructure：“任意 agent、任意 LLM，每个 sandbox 22 MB RAM，BYOC/on-prem，任意 file system。”

![Agent harness architecture diagram](https://pbs.twimg.com/media/HFaV5STa8AAp9kh.png)

[@HSVSphere](https://x.com/HSVSphere) 给出了一个[反向愿景](https://x.com/HSVSphere/status/2041480605632737774)：“未来没有特殊的‘agent sandbox’，未来是一个构建在动态语言之上的操作系统，对一切都有 capabilities 和（内在）scoping。” 讨论将其类比到 Plan9 和 Ruby，并把 TypeScript 斥为“too static”。

## 2. 令人困扰的问题

### 智能体安全和信任（Severity: High）

Skills 生态的增长速度已经超过安全基础设施。Skill Forge 审计发现，[88K+ 已发布 skills 中约 26% 存在安全漏洞](https://x.com/tom_doerr/status/2042003146885525856)。[@JamesonCamp](https://x.com/JamesonCamp) [警告](https://x.com/JamesonCamp/status/2041533186195574960)：“OpenClaw marketplace 中 12% 根本就是 malware。Keyloggers。身份盗窃。你的 AI 智能体正在安装陌生人发布的 packages。” [@conor_ai](https://x.com/conor_ai) [强调](https://x.com/conor_ai/status/2041901482052931876)信息泄露：“agents 可以、也会把敏感信息分享给任何正在和它们对话的人”，这促成 Hyperspell 采用受“Severance”启发的“innie/outie”智能体架构。

### 平台风险和创业公司焦虑（Severity: High）

Anthropic 发布 Managed Agents 引发了编排创业公司的生存焦虑。[@SolSt1ne](https://x.com/SolSt1ne) [捕捉到这种情绪](https://x.com/SolSt1ne/status/2041978378669138082)：“花 6 个月构建 custom agent infra、orchestration、retry logic、rate limiting，‘终于 production ready’，然后 Anthropic 发布 Claude Managed Agents public beta。” [@aakashgupta](https://x.com/aakashgupta) [指出](https://x.com/aakashgupta/status/2041940149328834748)，Anthropic 四天前刚阻止第三方 harness 使用订阅凭证，随后就发布了替代品。

### Context Window 管理不当（Severity: Medium）

[@gregisenberg](https://x.com/gregisenberg) [指出](https://x.com/gregisenberg/status/2042010418454184325)一个常见反模式：把 agent.md files 塞满指令，导致每次运行都消耗 7000+ token，拖累表现。“你越接近填满 context window，智能体表现就越差，就像别人一次性往你脑子里倒 10 件事时，你也会表现变差。”

### 智能体记忆丢失（Severity: Medium）

[@Mr_memsy](https://x.com/Mr_memsy) [指出](https://x.com/Mr_memsy/status/2041582460526379362)：“默认配置很快就会忘记。” 如果没有 temporal decay、embedding caches 和 governance files，智能体会“幻觉出过期 context”，而不是复利式积累知识。[@bellman_ych](https://x.com/bellman_ych) [描述](https://x.com/bellman_ych/status/2041767266094870890)了自己“凌晨 3 点读了 60MB leaked JS，到早餐时却在问‘oh-my-claudecode 又是什么来着’。”

## 3. 人们期望的功能

### Agent Sandbox Protocol

[@alexgshaw](https://x.com/alexgshaw) [发问](https://x.com/alexgshaw/status/2041946042040775167)：“谁在构建 Agent Sandbox Protocol？一种让用户指定 3P agent 在哪里执行 tools 的方式。” Anthropic 的博客描述了通过 tool calls 把“大脑”和“手”解耦，但还没有 vendor-neutral standard。

### 带验证的安全 Skills Marketplace

多个信号汇合：26% 的 skills 有漏洞，一些 marketplaces 中 12% 是 malware。Skill Forge 会扫描泄露的 API keys 并阻断 critical issues，但生态缺少一个可信、curated、按真实使用情况排名的 marketplace。[OKX 的 Plugin Store](https://x.com/wallet/status/2041757934632530100) 承诺“在一个地方搜索、安装和分享 Skills，并按真实使用情况排名”——但验证问题仍未解决。

### 长周期智能体可靠性

Gym-Anything benchmark [显示](https://x.com/wellecks/status/2041884780220883066)，最佳模型（GPT-5.4）在 500+ 步任务上的通过率只有 27.5%。[@tetsuoarena](https://x.com/tetsuoarena) 正在[构建 AgenC](https://x.com/tetsuoarena/status/2041589702919074283)，目标是“一个可以连续构建整整一年而不停下来的智能体”——但这个目标仍然偏愿景。

### 智能体原生身份与访问管理

[@conor_ai](https://x.com/conor_ai) 的 [innie/outie pattern](https://x.com/conor_ai/status/2041901482052931876) 是一种权宜方案。现在还没有标准，能根据智能体与谁交互来限定它能访问哪些数据。[@ashpreetbedi](https://x.com/ashpreetbedi) 用 [Dash 的 database-level RBAC](https://x.com/ashpreetbedi/status/2041653884536357122) 为数据智能体解决了这个问题，但这种方法是领域特定的。

**Dash v2** 的突出点是安全模型：Analyst 的 SQL connection 使用 `default_transaction_read_only=on`，Engineer 只写入 `dash` schema，eval suite 会明确尝试泄露凭证、执行破坏性 SQL，并跨越 schema 边界。学习循环同时存储 curated knowledge（validated queries、人类维护的 business rules）和 discovered knowledge（error patterns、gotchas，由智能体维护）。

**Gym-Anything** 解决了智能体评估中的根本缺口。当前 benchmarks 测试的是 consumer apps 上的短任务，但真实工作发生在专门的专业软件中——radiology tools、ERP systems、seismology monitors。CUA-World 通过让智能体构建智能体环境，相比标准部署降低 80-90% 成本，同时覆盖美国 22 个 SOC 职业组中的 200+ applications。

![Gym-Anything 论文摘要和领域覆盖](https://pbs.twimg.com/media/HFY7NiRW4AI11WJ.jpg)

## 6. 新动态与亮点

### AI 智能体发现真实漏洞

[@_colemurray](https://x.com/_colemurray) [披露](https://x.com/_colemurray/status/2041662615823458445)，他的 AI security agent “waclaude” 发现了 CVE-2026-1839——HuggingFace Transformers library 中由不安全 `torch.load()` 引发的任意代码执行。该漏洞是 PyTorch checkpoint loading 中的一行代码。这是智能体执行有效安全研究的具体证明，不只是生成报告。像 [PHALANX and METATRON](https://x.com/when_robots_cry/status/2041746829658718569) 这样的 security-focused agent harnesses（完全离线、本地 LLM）正在成为独立类别。

![CVE-2026-1839 Huntr report](https://pbs.twimg.com/media/HFVxVAoa8AMR-Wo.jpg)

### Retro Phone 在 London AI Engineer 接入语音智能体

[@ktoya_me](https://x.com/ktoya_me) [把 ElevenLabs voice agent 接到一部复古 rotary phone](https://x.com/ktoya_me/status/2041918230793458101)，放在 London AI Engineer conference 的红色英国电话亭里展示。这部电话是用 Claude 逆向工程出来的，并接入 ElevenLabs Agent。装置获得 64 个点赞和 5K+ 浏览量——它让参会者可以实际触摸和交互，成为语音智能体技术的实体演示。

## 7. 机会在哪里

**[+++] 强信号：Agent Skills 质量和安全基础设施。** 88K+ skills 已发布，26% 存在漏洞，一些 marketplaces 中 12% 是 malware。Skill Forge 解决单个 repos 的问题，但生态层面还没有一个带自动扫描、基于使用排名和 dependency analysis 的可信 registry。发布出来的东西和可以安全安装的东西之间的缺口，正比任何单一工具都更快扩大。

**[+++] 强信号：面向企业的 Managed Agent Infrastructure。** Anthropic 通过发布 Managed Agents 证明了需求。但企业需要 vendor-neutral options、multi-model support 和 compliance certifications。46% 的企业把 integration 列为主要智能体挑战。Notion 的集成模式——SaaS 作为编排层，模型提供商作为执行引擎——可能成为几十个垂直应用的模板。

**[++] 中等信号：长周期智能体评估和训练。** Gym-Anything 显示，最佳模型在现实的 500+ 步任务上失败率仍为 72.5%。能够以 80-90% 成本降低扩展到数千 replicas 的 agent training infrastructure，会打开 specialized evaluation-as-a-service providers 的市场。面向医疗、金融和工程软件的 domain-specific benchmarks 仍然大开。

**[++] 中等信号：智能体记忆和知识复利。** 当前智能体默认会遗忘。正在出现的方案——temporal decay、embedding caches、/dreaming modes、BRAID governance files——都碎片化且绑定框架。一个 cross-platform agent memory layer，如果能处理 persistence、relevance decay 和 multi-session compounding，将解决多个开发者提到的痛点。

**[++] 中等信号：语音智能体垂直解决方案。** LiveKit 的 [Rime Mist v3 integration](https://x.com/livekit/status/2041586877497508305) 通过音标括号展示了受监管行业需要的细粒度能力。[@yourealazyfvck](https://x.com/yourealazyfvck) [指出](https://x.com/yourealazyfvck/status/2041958942444761139)，voice agents 能以“同样价格”售出，但复杂度远低于 multi-tool automation。发音、合规和领域术语仍未在规模化场景下解决。

**[+] 新兴信号：Agent Sandbox Protocol 标准化。** 目前没有 vendor-neutral protocol 来指定智能体在哪里执行 tools。Anthropic 解耦大脑和手；AWS 提供 S3 Files；开源项目提供 22MB sandboxes。一个标准组织或事实协议如果能定义 agent execution environments，将降低碎片化。

**[+] 新兴信号：从 Trajectories 自动生成 Skill。** SkillX 展示了智能体能把自己的成功运行转换成可复用 skills。结合 [@gregisenberg](https://x.com/gregisenberg) 的递归 skill-building workflow，这指向能自我制造能力的 self-improving agent systems——目前仍处研究阶段，但生产应用清晰。

## 8. 要点总结

1. **Meta 带着原生多智能体编排重新进入前沿模型竞赛。** Muse Spark 的 Contemplating mode 可以在不成比例增加延迟的情况下跨智能体并行推理，“thought compression”降低 token 消耗。Benchmark 表显示它有竞争力但并不占优；long-horizon agentic coding 仍是缺口。（[source](https://x.com/AIatMeta/status/2041910285653737975)）

2. **Anthropic 的 Managed Agents 是一次威胁编排中间层的平台动作。** Anthropic 通过交付 production infrastructure，包括 fleet management、credential vaults 和 MCP integrations，把创业公司多年的 roadmap 压缩成一次发布。四天内先封锁 third-party auth，再发布替代品，显示出有意的生态整合。（[source](https://x.com/trq212/status/2041935792596304030)）

3. **Agent skills 是新的 package ecosystem，而且已经出现安全危机。** 88K+ 已发布 skills，26% 存在漏洞，一些 marketplaces 中 12% 是 malware。Skill Forge 和类似工具只能处理症状；生态缺少等同于 npm audit + verified publishers 的规模化机制。（[source](https://x.com/tom_doerr/status/2042003146885525856)）

4. **Context engineering 和 harness engineering 正在形式化为工程学科。** L1-L6 成熟度模型把从 prompt tricks 到 self-improving agents 的进展映射出来。OpenAI 的 Symphony 展示了终局：100 万 LOC、每天 10 亿 token、零人类代码或 review。这个终局和多数团队当前实践之间差距巨大。（[source](https://x.com/IntuitMachine/status/2041503368695185720)）

5. **真实智能体 benchmark 揭示了残酷限制。** Gym-Anything 的 CUA-World-Long 让智能体面对专业软件中的 500+ 步任务，最佳模型通过率只有 27.5%。Demo-quality agent performance 和真实 professional workflows 上的 production reliability 之间的差距，仍是这个领域的核心挑战。（[source](https://x.com/wellecks/status/2041884780220883066)）

6. **Skills over agent.md files 正成为共识性最佳实践。** 每次对话都加载完整 instruction files 会浪费 token 并降低表现。按需激活的 skills，加上递归改进 workflows，会产出更可靠的智能体。实践建议是：从手动 workflow 开始，让智能体写 skill，再围绕失败迭代。（[source](https://x.com/gregisenberg/status/2042010418454184325)）

7. **AI 智能体正在发现真实安全漏洞。** 一个 AI security agent 在 HuggingFace Transformers 中发现 CVE-2026-1839——真实的 arbitrary code execution bug。Security-focused agent harnesses（PHALANX、METATRON）正在成为一个专门类别，完全离线、使用本地 LLMs。Agent-driven security research 正从理论走向生产力。（[source](https://x.com/_colemurray/status/2041662615823458445)）
