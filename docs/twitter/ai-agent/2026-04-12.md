## 2. Pain Points: What Frustrates People

### Skill Sprawl Across Agents
**Severity: High | Prevalence: High**

The asm README captures it: "Skills scattered everywhere — `~/.claude/skills/`, `~/.codex/skills/`, `~/.openclaw/skills/`... you have the same skill installed three times and can't remember which version is where." With 17+ agent providers now supporting skills, practitioners spend significant time managing duplicates and incompatible formats. Coping: asm CLI (17 providers), npx skills add, Hermes skills browse. But these are still siloed per installer.

### Claude Code Perceived Quality Regression
**Severity: High | Prevalence: Moderate**

[@Hesamation](https://x.com/Hesamation)'s prompt forensics documented specific system prompt changes that may cause degradation. BridgeBench data showed Opus 4.6 dropping from #2 (83.3% accuracy) to #10 (68.3% accuracy) in hallucination benchmarks. [@rohit4verse](https://x.com/rohit4verse) (the day's top post with 1,793 likes) argued "coding is going away first" — but in replies, [@pchristou_](https://x.com/pchristou_) countered: "Coding isn't going anywhere. The human is still the contextual superpower." The tension between agent capability claims and regression reports creates anxiety.

### Agent Security Still an Afterthought
**Severity: High | Prevalence: Moderate**

[@TheNJDevOpsGuy](https://x.com/TheNJDevOpsGuy) captured the irony: "People: Don't use Agent Skills you downloaded from the internet, it's a security risk. People: Yeah, just `/plugin` in Claude or Codex and download/use any plugin from the internet." [@dbreunig](https://x.com/dbreunig) continued from 04-11: "finding bugs & exploits is the perfect marketing demo... An exploit agent in an isolated sandbox has a verification loop humans never need to review" (22 likes, [tweet](https://x.com/dbreunig/status/2043059745993728244)). Coping: asm's built-in security scanner, AgentAuditKit (77 security rules), and Snyk Agent Scan recommended by VoltAgent.

### MCP vs CLI Skills Tradeoff
**Severity: Moderate | Prevalence: Moderate**

[@MelkeyDev](https://x.com/MelkeyDev) flagged: "the MCP Abstraction Tax article outlines the trade-offs between using MCPs vs CLIs with skills. Every protocol layer between an agent and an API is a tax on fidelity" ([tweet](https://x.com/MelkeyDev/status/2043436546393821466)). [@jezell](https://x.com/jezell) was blunter: "If you control the code for both the agent and the backend services, 99.9% of the time you should not be using MCP. MCP solves one real problem" ([tweet](https://x.com/jezell/status/2043417554535010337)).

**Overall mood: Infrastructure consolidation with quality anxiety.**

The top post by engagement (1,793 likes, 3,953 bookmarks) is [@rohit4verse](https://x.com/rohit4verse) claiming "coding is going away first" — but the responses are measured and skeptical, highlighting the gap between bold claims and the reality of agents still needing significant harness engineering.
