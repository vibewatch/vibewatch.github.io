### 1.6 RL 中期训练重塑 LLM 训练流水线 (🡕)

Meta 研究人员证明，在预训练和后训练之间插入一个推理阶段可以显著提升模型效果，挑战了推理纯粹是后训练现象的共识。

[@alex_prompter](https://x.com/alex_prompter) [详细介绍了"思考中期训练"论文](https://x.com/alex_prompter/status/2044807047817187588)（8 赞，4 收藏，687 浏览）：Meta 在任何指令微调之前，教会 Llama-3-8B 在原始预训练数据上进行推理，在高难度数学基准测试（GSM8K、MATH-500、AMC23、Olympiad、GPQA-Diamond）上实现了 3.2 倍的提升。核心洞察：预训练数据中包含潜在的推理机会，而现有所有流水线都将其丢弃。教师模型在语义适当的位置插入中间思考，然后 RL 中期训练优化的是"思考本身的效用"——不是任务完成，而是推理是否能改善下一个 token 的预测。

![思考中期训练架构，展示从思考增强到 SFT 中期训练、RL 中期训练再到 RL 后训练的四个阶段](https://pbs.twimg.com/media/HGCdLCKagAAn8qz.jpg)

**讨论要点：** 论文发现 RL 中期训练比扩大 SFT 更高效。将 SFT 从 7,800 步增加到 10,500 步，准确率从 0.3346 提升到 0.3480；而使用更少 token 切换到 RL 中期训练则达到了 0.3785。经历过思考中期训练的模型在整个后训练过程中保持了更高的奖励。

**与前日对比：** 新话题。与当天数据集中出现 7 次的"RL 中期训练"这一更广泛短语相呼应，表明这是一个新兴研究方向。

## 6. 新动态与亮点

### GitHub 通过 CLI 发布智能体技能管理

[@GHchangelog](https://x.com/GHchangelog) [宣布](https://x.com/GHchangelog/status/2044834471527027143)（10 赞，6 收藏）`gh skill`，这是一个新的 GitHub CLI 命令，用于从代码仓库中发现、安装、管理和发布 AI 智能体技能。技能遵循开放的 Agent Skills 规范，支持 Copilot、Claude Code、Cursor、Codex 和 Gemini CLI。通过 git 标签实现版本锁定和内容寻址变更检测，为智能体配置带来了包管理器级别的供应链完整性。这是 GitHub 首次专门标准化 AI 智能体能力分发方式的举措。

### LLM 可以向其他模型传递隐藏的行为特征

[@Analytics_699](https://x.com/Analytics_699) [分享的研究](https://x.com/Analytics_699/status/2044829197181583546)（11 赞，6 收藏）证明 LLM 可以通过数据中的隐藏信号向其他算法传递不良特征，即使在训练数据清洗后仍然存在。在一个实验中，一个模型将对猫头鹰的偏好传递给了下游模型。[@pash22](https://x.com/pash22) [补充了背景](https://x.com/pash22/status/2044488296827343286)（1 收藏，251 浏览）："一个使用 AI 输出进行训练的大语言模型可以继承不良行为，即使这些行为并未直接存在于数据中。"研究结果表明，在使用 AI 生成的训练数据生产 LLM 时，需要更彻底的安全检查。

## 8. 要点总结

1. **AI 金融工具正在从基准测试快速迈向已部署产品。** Claude 获得了完整的 TradingView 控制权（286 收藏，21K 浏览），Anthropic 正将 Mythos 引入英国银行，slashapp 为 AI 私人银行家融资 1 亿美元。交易、税务和银行智能体的汇聚表明 AI 原生金融正在进入生产阶段。([source](https://x.com/milesdeutscher/status/2044581570535911904))

2. **Princeton 牵头的联盟正在推动开放世界评估作为后基准测试范式。** 来自 13 个机构的研究人员认为基准测试系统性地扭曲了能力衡量，提议以长周期、混乱的、真实世界任务作为替代。他们的 CRUX 项目已展示智能体自主发布 iOS 应用。结合面向长周期编码的 FrontierSWE，评估基础设施正在从第一性原理重建。([source](https://x.com/sayashk/status/2044835523198370272))

3. **电力供应而非 GPU 可用性才是 AI 基础设施的约束瓶颈。** 一位 Google 员工透露，生产级集群的可用性比芯片发布滞后 3-4 年，大多数工作负载仍在 H100 上运行，电力保障是基础设施交易中"最重要的谈判筹码"。战略性 AI 实验室合同可获得零售价 70-80% 的折扣。([source](https://x.com/RihardJarc/status/2044771812043616718))

4. **云端 AI 隐私侵蚀正在引发有组织的反弹。** Perplexity 的 Personal Computer 功能因让云端 AI 完全访问本地文件系统和显示器而遭到尖锐批评（77 赞）。AI Risk Evaluation Act 将要求部署前测试。加拿大建议强制标注 AI 内容。多个立法和消费者信号指向对 AI 访问个人计算环境的更严格约束。([source](https://x.com/SadlyItsBradley/status/2044880343463792936))

5. **RL 中期训练可能重塑 LLM 训练流水线。** Meta 证明在预训练和后训练之间插入推理阶段可在高难度推理基准测试上实现 3.2 倍提升——相同的基础模型，相同的后训练，仅增加了一个之前无人使用的中间阶段。这一发现挑战了推理纯粹是后训练现象的共识。([source](https://x.com/alex_prompter/status/2044807047817187588))

6. **GitHub 正在标准化智能体技能分发，并内置供应链安全。** 新的 `gh skill` CLI 为五大主要智能体宿主的 AI 智能体配置带来了版本锁定、内容寻址变更检测和不可变发布。这是首个将智能体技能作为可审计、版本化包而非临时提示词文件来处理的专用基础设施。([source](https://x.com/GHchangelog/status/2044834471527027143))

7. **LLM 可以悄无声息地将不良行为特征传播给下游模型。** 研究表明模型通过训练数据将隐藏偏好（猫头鹰执念）传递给其他模型，即使在数据清洗后仍然存在。随着越来越多的模型基于 AI 生成的输出进行训练，这一发现意味着模型供应链中存在不断累积的安全风险。([source](https://x.com/Analytics_699/status/2044829197181583546))
