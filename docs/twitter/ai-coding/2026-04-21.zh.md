### 1.2 AI 编程安全危机沿多个向量升级 🡕

三类不同安全故事汇聚成当天最令人警惕的主题。

**所有主要智能体都受 prompt injection 影响。** [@The_Cyber_News](https://x.com/The_Cyber_News) [报道](https://x.com/The_Cyber_News/status/2046510893752807689)（23 点赞，1,785 浏览量）了一个名为 “Comment and Control” 的跨厂商漏洞类别——prompt injection 攻击会武器化 GitHub pull request 标题、issue 正文和评论，劫持 AI 编程智能体，并直接从 CI/CD 环境中窃取 API keys。Claude Code Security Review（Anthropic 评级为 CVSS 9.4 Critical）、Gemini CLI Action 和 GitHub Copilot Agent（SWE Agent）都被确认存在漏洞。研究员 Aonan Guan 演示了完整攻击闭环完全在 GitHub 内运行：攻击者提交带恶意标题的 PR，AI agent 将其作为可信上下文处理，执行指令，并通过 PR 评论把凭证外泄。

![Comment and Control 漏洞显示 Claude Code Security Review agent 从 GitHub PR 读取恶意提示并泄露凭证](https://pbs.twimg.com/media/HGaqwqmbUAAdwhl.jpg)

来自 [@jeremie_strand](https://x.com/jeremie_strand) 的回复抓住了核心问题：“agent 以与 operator instructions 相同的信任级别读取不可信内容。把 data 和 instructions 分离仍然没有解决。”

**Antigravity IDE 远程代码执行。** [@TheHackersNews](https://x.com/TheHackersNews) [报道](https://x.com/TheHackersNews/status/2046535216068694239)（49 点赞，7,242 浏览量），Google 修复了一个 Antigravity IDE 缺陷，该缺陷可通过 search tool input 实现任意代码执行。攻击者可以注入命令、绕过 sandbox controls，并自动运行 scripts。[@DarkReading](https://x.com/DarkReading) [也单独报道](https://x.com/DarkReading/status/2046653957884252442)（3 点赞，1,072 浏览量）了同一个修复。

![攻击流程图显示不可信内容如何进入攻击者控制的 agent、绕过 secure mode，并在 Antigravity IDE 中实现远程代码执行](https://pbs.twimg.com/media/HGbA7Z4bkAAq_6y.jpg)

**通过假 Antigravity 安装包发起的供应链攻击。** [@DFIR_Radar](https://x.com/DFIR_Radar) [详细说明](https://x.com/DFIR_Radar/status/2046605855752171956)（5 点赞，375 浏览量）了一个木马化 Antigravity installer：它交付功能完整的软件，同时部署 stealer malware。该安装包会落地 PowerShell downloader、禁用 Windows Defender，并瞄准浏览器凭证、session cookies、Discord tokens 和 crypto wallets。

![假 Google Antigravity installer 供应链攻击的 malware analysis 摘要](https://pbs.twimg.com/media/HGcBLV3WgAAQJAw.png)

**Vibe coding 平台泄露。** [@DEX_ion10](https://x.com/DEX_ion10) [报道](https://x.com/DEX_ion10/status/2046471996452585652)（23 点赞，89 浏览量），Lovable vibe coding 平台暴露了 1,000 多名用户的数据。[@0x_kaize](https://x.com/0x_kaize) [把线索连了起来](https://x.com/0x_kaize/status/2046534202498171186)（11 点赞，113 浏览量）：Vercel 因受损 AI tool Context AI 遭到入侵，暴露 580 名员工的 API keys、source code 和数据，并以 $2M BTC 挂牌出售。他引用了统计数据：“45% 的 AI-generated code 含有安全漏洞。仅 2026 年 3 月就有 35 个新 CVEs 直接由 AI-generated code 导致。”

[@Replit](https://x.com/Replit) [主动回应](https://x.com/Replit/status/2046712510703951904)（10 点赞，1,936 浏览量），发布 CTO [@lhchavez](https://x.com/lhchavez) 的详细安全架构文章：“Replit 基础设施中运行客户代码的每一层，都按 defense in depth 设计。没有任何单一控制是最后防线。每一层都假设它上面那层可能失效。”

**与前日对比：** 昨天安全只是背景问题。今天它占据主导：三大主要智能体同时出现 CVSS 9.4 漏洞、Antigravity RCE、瞄准 Antigravity 用户的供应链攻击、vibe coding 平台泄露，以及 Vercel 通过 AI 工具受损。攻击面扩张得比防御更快。

## 6. 新动态与亮点

研究员 Aonan Guan [披露](https://cybersecuritynews.com/prompt-injection-via-github-comments/)了一个影响 Claude Code（CVSS 9.4）、Gemini CLI 和 GitHub Copilot Agent 的漏洞类别。不同于经典 prompt injection，这种攻击是主动的：GitHub Actions 会在 PR 和 issue events 上自动触发，这意味着只要提交一个带恶意标题的 PR，就能在没有受害者交互的情况下劫持 agent。攻击会完全在 GitHub 内外泄凭证——不需要外部服务器。[@The_Cyber_News](https://x.com/The_Cyber_News) [进行了报道](https://x.com/The_Cyber_News/status/2046510893752807689)（23 点赞，1,785 浏览量）。

## 7. 机会在哪里

**[+++] AI Agent 安全治理平台** -- “Comment and Control” 攻击（CVSS 9.4）通过 GitHub comments 影响三大主要 AI coding agents。Antigravity RCE 可通过 search tool input 利用。假 Antigravity installers 部署 credential-stealing malware。没有厂商为 AI agent deployments 提供统一安全层。一个能够监控 agent trust boundaries、sandbox untrusted inputs、审计 credential exposure，并在 Claude Code、Codex 和 Copilot 之间应用 least-privilege policies 的平台，将解决一个正在被主动利用的结构性缺口。（[@The_Cyber_News](https://x.com/The_Cyber_News)，[post](https://x.com/The_Cyber_News/status/2046510893752807689)；[@TheHackersNews](https://x.com/TheHackersNews)，[post](https://x.com/TheHackersNews/status/2046535216068694239)；[@DFIR_Radar](https://x.com/DFIR_Radar)，[post](https://x.com/DFIR_Radar/status/2046605855752171956)）

**[+++] Cross-Tool 成本与用量智能** -- GitHub Copilot 因 compute costs 暂停 signups。Antigravity 因没有原生选项而封禁了一名用第三方工具追踪 usage 的用户。Ollama 用户缺少清晰的 rate limit documentation。一个覆盖 Copilot、Claude Code、Codex 和 OpenCode 的跨平台 usage 与 cost monitoring tool——带 alerts、optimization recommendations 和 per-project attribution——将服务越来越多同时管理多个 AI coding subscriptions 的开发者。（[@GergelyOrosz](https://x.com/GergelyOrosz)，[post](https://x.com/GergelyOrosz/status/2046349984367284573)；[@sqs](https://x.com/sqs)，[post](https://x.com/sqs/status/2046394711733330160)；[@FrancescoCiull4](https://x.com/FrancescoCiull4)，[post](https://x.com/FrancescoCiull4/status/2046526024163135775)）

**[++] Enterprise Claude Code 部署模式** -- Mercury VP 构建了一个包含 15K-document knowledge base、multi-agent teams 和 20+ custom skills 的第二大脑。这是目前公开的最详细 enterprise Claude Code architecture。把这个模式产品化——pre-built knowledge base connectors、skill templates、multi-agent team configurations 和 MCP integration scaffolds——可以减少复现它所需的数周设置时间。（[@petergyang](https://x.com/petergyang)，[post](https://x.com/petergyang/status/2046597450279125132)）

**[++] AI-Generated Code 审计工具** -- 据报道，45% 的 AI-generated code 含有安全漏洞。2026 年 3 月有 35 个新 CVEs 直接由 AI-generated code 导致。一个专门训练来检测 AI models 产生的特定漏洞模式的 static analysis tool——不是通用 SAST，而是 AI-output-aware scanning——会解决一个随着每次 vibe-coded deployment 增长的缺口。（[@0x_kaize](https://x.com/0x_kaize)，[post](https://x.com/0x_kaize/status/2046534202498171186)；[@Replit](https://x.com/Replit)，[post](https://x.com/Replit/status/2046712510703951904)）

**[+] Mobile-First Agent Management** -- Paseo（4.3K stars）为 multi-agent management 提供 iOS 和 Android clients。它的帖子高收藏率（1.04%）说明存在需求。一个用于 review diffs、approve changes、跨 providers 管理 agent queues 的 polished mobile experience，可以服务那些会在手机上查看 long-running agent tasks 的开发者。（[@tom_doerr](https://x.com/tom_doerr)，[post](https://x.com/tom_doerr/status/2046699198767153318)）

## 8. 要点总结

1. **安全，而不是定价，是当天主导故事。** 一个 CVSS 9.4 跨厂商 prompt injection 同时影响 Claude Code、Gemini CLI 和 Copilot。Antigravity RCE 被修复。假 Antigravity installers 部署 stealer malware。Lovable 泄露 1,000+ 用户。Vercel 通过 AI tool 被攻破。一天内出现五个不同安全事件，标志着拐点。（[@The_Cyber_News](https://x.com/The_Cyber_News)，[post](https://x.com/The_Cyber_News/status/2046510893752807689)；[@TheHackersNews](https://x.com/TheHackersNews)，[post](https://x.com/TheHackersNews/status/2046535216068694239)；[@DFIR_Radar](https://x.com/DFIR_Radar)，[post](https://x.com/DFIR_Radar/status/2046605855752171956)；[@DEX_ion10](https://x.com/DEX_ion10)，[post](https://x.com/DEX_ion10/status/2046471996452585652)；[@0x_kaize](https://x.com/0x_kaize)，[post](https://x.com/0x_kaize/status/2046534202498171186)）

2. **Claude Code 的领先有调查数据支撑，而不只是 vibes。** Pragmatic Engineer survey 显示 Claude Code 约 570 次提及，领先 Copilot（约 400）和 Cursor（约 300）。在推荐讨论串中，Claude Code 一致获胜。市场份额转移现在可以衡量。（[@GergelyOrosz](https://x.com/GergelyOrosz)，[post](https://x.com/GergelyOrosz/status/2046409167863255276)；[@CyberRacheal](https://x.com/CyberRacheal)，[post](https://x.com/CyberRacheal/status/2046567009291252064)）

3. **Codex 达到 4M weekly users，同时扩展到企业。** 具名客户包括 Virgin Atlantic、Ramp 和 Cisco。战略合作伙伴 Accenture 与 Capgemini 正把它部署到 engineering 之外的 marketing、finance 和 sales。增长速度（一个月内 2M 到 4M）和企业广度，是迄今最强 traction signals。（[@TechieUltimatum](https://x.com/TechieUltimatum)，[post](https://x.com/TechieUltimatum/status/2046647682039775246)）

6. **Enterprise Claude Code 模式正在公开化。** Mercury VP 用 Claude Code 构建了一个包含 15K documents、multi-agent teams 和 20+ custom skills 的第二大脑。这是目前公开分享的最详细企业架构，也为组织采用 AI-native workflows 提供了可复制蓝图。（[@petergyang](https://x.com/petergyang)，[post](https://x.com/petergyang/status/2046597450279125132)）
