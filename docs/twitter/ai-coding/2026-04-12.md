### 1.6 Agent Security Scanning Tooling (↑ emerging)

[@Sattyamjjain](https://x.com/Sattyamjjain) launched [AgentAuditKit](https://github.com/sattyamjjain/agent-audit-kit) — a security scanner for MCP-connected AI agent pipelines with 77 rules, 13 scanners, and OWASP Agentic Top 10 coverage (10/10). The tool scans Claude Code, Cursor, Copilot, and Windsurf configs for hardcoded secrets, shell injection, tool poisoning, and rug pulls (0 likes but 4 replies, [tweet](https://x.com/Sattyamjjain/status/2043337334700286011)). Every AI coding assistant uses MCP now. Nobody is auditing the configs."

The tool ships as a GitHub Action, CLI, pre-commit hook, Docker container, and VS Code extension. GitHub repo has 1 star and 0 forks (very early), but the feature set is comprehensive: SHA-256 tool pinning for rug pull detection, Python AST taint analysis tracing `@tool` parameter flows to dangerous sinks (eval, subprocess, SQL), and compliance mapping to EU AI Act, SOC 2, ISO 27001, HIPAA, and NIST AI RMF.

**Discussion insight:** The zero-like, four-reply pattern suggests a technically competent builder shipping to a small audience. The MCP CVE data (30 in 60 days) provides concrete evidence that AI coding agent security is a real and quantifiable problem, not theoretical.
