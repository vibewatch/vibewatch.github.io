### 1.1 SoulForge: Codebase-Aware Agent Architecture (↑ emerging)

The day's dominant tweet by a massive margin: [@BniWael](https://x.com/BniWael) launched SoulForge — an AI coding agent that builds a live graph of the codebase before the agent reads a single line of code, achieving 1.8x faster completion and 2.1x lower cost than Claude Code, Codex, and OpenCode on the same bug with the same Claude Opus 4.6 model. The tweet pulled 524 likes, 57K views, and 834 bookmarks — the highest bookmark count of the day ([tweet](https://x.com/BniWael/status/2042364421373121018)). SoulForge's core thesis: "Everyone's fighting over Claude Code vs Codex vs OpenCode. Meanwhile their tokens are burning because the agent is grep-ing the entire repo and praying on every prompt, with 0 architectural knowledge of the codebase."

The technical differentiation is substantial: real LSP powered by an embedded Neovim editor (go-to-definition, references, call hierarchy, type info), surgical reads instead of dumping entire files into context, multi-tab editing with cross-tab coordination through a real-time bus, and per-agent model routing — Opus as the brain, Gemini for research, Haiku for quick edits, Ollama for local throwaway tasks. It supports 30+ languages, 20+ providers, and offers $0 compaction.

**Discussion insight:** The 834 bookmarks against 524 likes — a 1.6:1 bookmark-to-like ratio — signals exceptional "I need to try this" intent. [@yabsssai](https://x.com/yabsssai) probed the methodology: "what kind of architectural knowledge of the codebase do you think would make a significant difference in this scenario?" — questioning whether the gains are from the graph itself or simply from more targeted file reads.

[@theo](https://x.com/theo) asked: "I want to control my agents from my phone. I'm sure you do too. What workflows do you want?" [@MATEOINRL](https://x.com/MATEOINRL) and [@carlsue](https://x.com/carlsue) are manually arbitraging models, but there's no dashboard that shows cost-per-task across Codex, Claude Code, and Copilot. **Functional need, nice-to-have.** Opportunity: 🟡 competitive.

The need: session state that follows you across agents and providers. **Functional need, must-have.** Opportunity: 🟡 competitive.
