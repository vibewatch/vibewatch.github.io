### 1.1 Claude Code 桌面端重新设计与 Routines 发布（🡕）

Anthropic 在同一天为 Claude Code 发布了两项重大更新。官方账号 [@claudeai](https://x.com/claudeai) [宣布了桌面端的全新设计](https://x.com/claudeai/status/2044131493966909862)，支持并行会话并排运行、新增侧边栏进行管理、集成终端、文件编辑、HTML/PDF 预览以及拖放式布局。[@9to5mac](https://x.com/9to5mac) [报道了此次发布](https://x.com/9to5mac/status/2044133564384502179)，并揭示了推文中未提及的第二个功能：Routines——在 Anthropic 网页基础设施上运行的定时自动化任务，即使用户的 Mac 处于离线状态也能执行。[9to5Mac 文章](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/)详细介绍了不同层级的限制：Pro 每天 5 个 Routines，Max 每天 15 个，Team/Enterprise 每天 25 个。Routines 支持仓库和连接器访问，省去了用户自行管理 cron 任务和 MCP 服务器基础设施的需要。

[@Yuchenj_UW](https://x.com/Yuchenj_UW) [将此次重新设计定义为](https://x.com/Yuchenj_UW/status/2044133573326934384) IDE 范式转变，引用了 Andrej Karpathy 的话："我们需要一个更大的 IDE。基本单位不再是文件，而是智能体。"其含义是："Cursor 现在必须为定义 IDE 的未来而战。"[@RoundtableSpace](https://x.com/RoundtableSpace) [以当天最高浏览量](https://x.com/RoundtableSpace/status/2044185178063679580)（4,898 次浏览，13 次收藏）放大了这一热度，引来 [@traleyweb3](https://x.com/traleyweb3) 等用户回复："Claude Code 刚刚迎来了一次重大升级。"

**讨论要点：** 重新设计引发了广泛传播，但没有收到技术层面的批评。Routines 功能——服务端定时自动化——是结构上更为重要的更新，但在社交媒体上获得的关注度却不及视觉重新设计。

**与前日对比：** 昨日报告追踪了 Copilot CLI 走向远程和移动端，以及 Codex 转型为桌面超级应用。今天，Claude Code 通过让桌面应用支持多会话并添加服务端自动化来回应——在追随"设备无关"方向的同时，增加了两个竞争对手都尚未宣布的调度能力。

两篇帖子展示了可安装技能生态系统的持续增长，这些技能扩展了 Claude Code 的开箱即用知识。

另外，[@hasantoxr](https://x.com/hasantoxr) [介绍了 Taste Skill](https://x.com/hasantoxr/status/2044008552231252248)，这是一个开源的智能体技能集合，赋予 AI 编程工具"设计品味感"。该[仓库](https://github.com/LeonxLnx/taste-skill)拥有 8.3k 星标和 827 次 fork，包含七种技能变体（taste-skill、redesign-skill、soft-skill、brutalist、minimalist、stitch、output-skill）和三个可调旋钮，分别控制布局变化度、动效强度和视觉密度。与 SparkJS 技能不同，Taste Skill 支持跨工具使用：Cursor、Claude Code、Codex、Windsurf、Copilot 和 Antigravity。

**讨论要点：** SparkJS 技能展示了领域特定知识注入（3D 图形 API），而 Taste Skill 展示了跨领域质量注入（设计模式）。两者共同呈现了两种不同的技能模式：深度垂直专业能力 vs. 广度水平质量提升。

## 2. 令人困扰的问题

### Claude Code 成本不透明（中等）

[@GithubAwesome](https://x.com/GithubAwesome) [直接描述了这个问题](https://x.com/GithubAwesome/status/2044203308642968037)："有人每天在 Claude Code API 额度上花费超过 200 美元，却完全不知道钱花在了哪里。"解决方案是 codeburn，一个终端 UI 工具，它读取本地会话记录并按任务类型（调试、头脑风暴、编码）对每个 token 的花费进行分类——将"来自臃肿 MCP 服务器的 token 开销"识别为成本驱动因素。该工具包含一个交互式每日活动图表和一个 Mac 菜单栏小组件。分析过程无需任何 LLM 调用。

这延续了昨日报告中的 token 膨胀困扰，当时已确认 Claude Code v2.1.100+ 每次请求注入约 20k 服务端 token，系统开销在用户消息发出前就消耗了约 50k token。今天的证据新增了一个成本因素：臃肿的 MCP 服务器加剧了 token 浪费。

### Claude Code 用户流失信号（低）

[@inancgumus](https://x.com/inancgumus) [简单表示](https://x.com/inancgumus/status/2044202972578467949)："我已经完全抛弃了 Claude Code。"该帖子获得 374 次浏览，但零赞和零收藏——关注者看到了但没有传播。帖子未给出转换的原因。作为一个孤立数据点，这代表了一个缺乏诊断价值的流失信号，但它处于多日以来记录的成本和质量问题的大背景之下。

## 3. 人们期望的功能

### 精细化 Token 成本归因

"每天 200 美元的 Claude Code API 额度"（[@GithubAwesome](https://x.com/GithubAwesome)）与任何内置成本可见性之间的差距依然很大。codeburn 通过读取本地会话记录来解决这个问题，但该方案是外部的、事后的。用户需要内置于智能体中的实时、按任务的成本归因——在开始之前就知道一次调试会话的成本是代码生成会话的 3 倍，而不是在账单到来之后才发现。昨日报告标记了同样的需求；今天的数据以一个具体的金额（每天 200 美元）加以印证。

### 跨工具技能可移植性

Taste Skill 的跨工具兼容性（Cursor、Claude Code、Codex、Windsurf、Copilot、Antigravity）凸显了大多数技能所缺乏的特性：可移植性。SparkJS 技能仅适用于 Claude Code。随着技能生态系统的增长，开发者将希望一次安装、到处使用——一种跨智能体通用的技能注册表和格式，类似于 MCP 标准化工具连接的方式。Taste Skill 使用的 `npx skills add` 命令暗示了这一方向。

## 6. 新动态与亮点

### Claude Code Routines：服务端定时自动化

Anthropic 以研究预览形式推出了 Routines——在 Claude Code 网页基础设施上运行的定时自动化任务，可访问仓库和连接器。这将 Claude Code 从一个被动工具（用户提示词，智能体响应）转变为一个主动工具（智能体按计划运行，无需用户在场）。各层级限制：Pro 每天 5 次，Max 每天 15 次，Team/Enterprise 每天 25 次。（[9to5Mac](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/), [帖子](https://x.com/9to5mac/status/2044133564384502179)）

### Claude Code 桌面端支持多会话

重新设计的 Claude Code 桌面应用支持在单个窗口中运行多个并行会话，并配有管理侧边栏。新增集成终端、文件编辑、HTML/PDF 预览以及拖放式布局中更快的 diff 查看器。这直接解决了开发者在代码库不同部分同时运行多个智能体任务的工作流需求。（[帖子](https://x.com/Yuchenj_UW/status/2044133573326934384)）

### SparkJS Skill：首个面向 3D 图形的领域特定 Claude Code 技能

一个 MIT 许可的技能，赋予 Claude Code 对 SparkJS 3D 高斯泼溅渲染库的完整知识——涵盖渲染器、泼溅网格、SDF 编辑、细节层次流式加载以及 117+ 着色器标准库函数。通过单条 git clone 命令安装。展示了将领域特定 API 知识打包为可移植技能文件的模式。（[GitHub](https://github.com/shi3z/sparkjs-skill), [帖子](https://x.com/shi3z/status/2044202378140995643)）

## 7. 机会在哪里

**[+++] 智能体技能注册表与跨工具可移植性** — SparkJS Skill（仅限 Claude Code）和 Taste Skill（6+ 工具）展示了两种同时涌现的技能分发模型。Taste Skill 的 `npx skills add` 和跨工具支持拥有 8.3k 星标；SparkJS 的仅限 Claude 的 git clone 方式在数小时内获得了 22 个赞和 12 次收藏。一个统一的技能注册表——技能发布一次即可在 Claude Code、Cursor、Codex、Windsurf 和 Copilot 上安装——将获得与 MCP 在工具连接领域同等的生态系统价值。需求已经存在；标准尚未建立。

**[++] AI 编程智能体的实时成本归因** — 昨日报告记录了用户消息发出前约 50k token 的系统开销。今天的数据补充了每天 200 美元的成本数据和 codeburn 作为首个外部解决方案。机会在于将成本归因内置到智能体本身：执行前的按任务成本估算、预算警报以及按成果的成本指标。每一位 Claude Code API 用户都是这一能力的潜在客户。

**[++] 超越 Anthropic 的定时智能体自动化** — Claude Code Routines 是首个面向 AI 编程智能体的服务端定时自动化功能。该功能受层级限制（每天 5-25 次）且为 Anthropic 独占。一个开源替代方案——支持任何 LLM 后端、在开发者自有基础设施上运行的定时智能体任务——将满足昨天推动 Multica 达到 4,000+ 星标的同一需求（自托管智能体基础设施）。

**[+] AI 辅助教育平台** — Ship Your First App 展示了 Claude Code 作为面向非技术用户的结构化辅导平台。其课程模式（获取指令、指导构建、交互练习）可以从 Web 开发推广到任何技术领域。差距在于目前仅有第一阶段内容——模块 1-3 和阶段 2-3 均标记为"即将推出"。率先在多个领域推出完整课程的先行者将占据这一品类。

## 8. 要点总结

1. **Claude Code 在同一天发布了桌面端重新设计和服务端 Routines。** 并行会话、集成终端和拖放式布局更新了桌面体验。Routines——运行在 Anthropic 基础设施上的定时自动化——将 Claude Code 从被动变为主动，并设有层级限制（每天 5-25 次）。（[9to5Mac](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/)）

2. **Claude Code 技能生态系统正以两种不同模式成型。** SparkJS Skill（领域特定 API 知识，仅限 Claude）和 Taste Skill（跨领域设计质量，8.3k 星标，6+ 工具）展示了垂直深度与水平广度。跨工具可移植性是核心差异化因素。（[SparkJS](https://x.com/shi3z/status/2044202378140995643), [Taste Skill](https://x.com/hasantoxr/status/2044008552231252248)）

3. **Claude Opus 4.7 的猜测有发布节奏分析的支撑。** 距 Opus 4.6 已过 68 天，社区已处于预测的发布窗口期内。据报道，源代码泄露确认了 Opus 4.7 和 Sonnet 4.8 的模型名称。Opus 4.6 被感知的性能削弱遵循了与 Opus 4.5 淘汰周期相同的模式。（[帖子](https://x.com/bridgemindai/status/2044007189107331487)）

4. **Claude Code 成本不透明现在有了具体数字：每天 200 美元且毫无可见性。** codeburn 通过读取会话记录在外部解决了这一问题，但真正的需求是内置的实时成本归因。臃肿的 MCP 服务器被新识别为一个成本驱动因素，与昨日记录的 token 膨胀并列。（[帖子](https://x.com/GithubAwesome/status/2044203308642968037)）

5. **Claude Code 正从编程扩展到交易、教育和投资人研究。** 一个四阶段加密货币代币筛选器、一个交互式 Web 开发导师以及一位日本投资人的背书，都将 Claude Code 定位为一个通用智能体平台，而非仅仅是编程工具。（[交易](https://x.com/shanesimpson513/status/2044203657416368576), [教育](https://x.com/hot_town/status/2044070979534557424), [投资人](https://x.com/kskgroup2017/status/2043960844414017959)）
