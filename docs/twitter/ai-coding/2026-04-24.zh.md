## 8. 要点总结

1. **GPT-5.5的采用正在驱动真实的订阅变更，而不仅是基准测试热潮。** [@mckaywrigley](https://x.com/mckaywrigley) [报告](https://x.com/mckaywrigley/status/2047806060061507971)在不到三个月内从80/20的Claude/GPT比例翻转为80/20的GPT/Claude比例。[@Kappaemme1926](https://x.com/Kappaemme1926) [晒出了Claude取消订阅邮件](https://x.com/Kappaemme1926/status/2047598921996984495)。[@Colewherld](https://x.com/Colewherld) [称Codex](https://x.com/Colewherld/status/2047684720675221653)"比Claude Code好了不止一个档次。"次日信号是迁移，而非仅仅是评估。

2. **速率限制是所有提供商共同面临的关键用户体验问题。** [@notjazii](https://x.com/notjazii) [在36小时内耗尽了](https://x.com/notjazii/status/2047616448818708828)Antigravity配额（90条回复，数据集最高）。[@MilkRoadAI](https://x.com/MilkRoadAI) [报道](https://x.com/MilkRoadAI/status/2047797528293188024)Anthropic限制了Claude的思考深度。2026年超大规模云厂商合计7000亿美元的资本支出仍无法满足需求。算力短缺是结构性的，而非暂时的。

3. **OpenCode已成为评估新模型的中立阵地。** [@rileybrown](https://x.com/rileybrown) [向4,998人提问](https://x.com/rileybrown/status/2047734309361254897)"测试新DeepSeek模型的最佳方式是什么？"答案是OpenCode。[@mehulmpt](https://x.com/mehulmpt) [用它](https://x.com/mehulmpt/status/2047778103645364295)进行了四方模型对比。[@opencode](https://x.com/opencode) [在发布后数小时内集成了](https://x.com/opencode/status/2047562491878572153)DeepSeek V4。随着各实验室垂直整合（Codex、Claude Code、Gemini Code Assist、Cursor/xAI），模型无关层变得更加关键。
