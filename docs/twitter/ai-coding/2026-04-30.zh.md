### 1.4 Codex Desktop App：与 Claude Code 的功能对比 🡒

[@aniketapanjwani](https://x.com/aniketapanjwani) [发布](https://x.com/aniketapanjwani/status/2049661842373451855)（19 点赞，29 收藏数，677 浏览量）一份详细 10 点对比，来自录制 4+ 小时 Codex Desktop 课程后的体验：

- OpenAI 的界面“明显高于 Anthropic”——plugins 很直观，Claude 的 Connectors 让人困惑
- Codex 限制 6 个 subagents；Claude Code 支持最多 16 个
- Codex subagents 必须显式调用；Claude Code 会自动调用
- Codex plugins 不能包含 subagents

- Auto-review permission mode “效果相当不错”——比 Claude Code 的 auto mode 更好
- Cloud agents 限制在 gpt-5.3-codex，且没有 infrastructure-as-code 环境设置

## 3. 人们期望的功能

### Token 成本可见性和优化成为默认能力

[@InduTripat82427](https://x.com/InduTripat82427) [整理](https://x.com/InduTripat82427/status/2049894134417326432)10 个 token reduction repos（RTK: 60-90%，Context Mode: 98%，Token Savior: 97%）。开发者需要自己发现、安装并叠加 2-3 个独立工具来管理 token 消耗，说明这应该由 agents 自带。没有编程智能体提供原生 token budgeting 或 pre-task cost estimation。

Urgency: High. Opportunity: Direct -- GitHub Copilot usage-based billing 将于 6 月 1 日生效。

### Codex subagent 与 Claude Code 看齐

[@aniketapanjwani](https://x.com/aniketapanjwani) [指出](https://x.com/aniketapanjwani/status/2049661842373451855)三个缺口：Codex 限制 6 个 subagents（Claude Code: 16）、subagents 需要用户显式调用（不会自动 spawn）、plugins 不能包含 subagents。对能从并行分析中获益的 code review 工作流而言，6-agent cap 是主要瓶颈。

Urgency: Medium. Opportunity: Direct -- 这由 OpenAI 控制；power user 反馈可能加速改进。

### 面向智能体工作流的 worktree 环境引导

[@aniketapanjwani](https://x.com/aniketapanjwani) [指出](https://x.com/aniketapanjwani/status/2049661842373451855)：“如果 Codex 能内置一些用于引导 worktree 环境的 skill 就好了——类似 Claude Code 用 /update-config 来初始化仓库权限。”他构建了自己的 `/worktree-cli-bootstrap` skill，但认为这种摩擦阻碍了 worktree 的采用。

Urgency: Medium. Opportunity: Indirect -- 社区 skill 可以在官方支持前解决。

## 7. 机会在哪里

**[+++] Token Budget Management as a Native Agent Feature** -- [@InduTripat82427](https://x.com/InduTripat82427) [整理](https://x.com/InduTripat82427/status/2049894134417326432)的 10 个 token reduction repos（RTK：终端输出省 80%，Context Mode：SQLite sandboxing 省 98%，Token Savior：导航省 97%）说明，token management 目前分散在社区工具里。GitHub Copilot usage-based billing 将于 6 月 1 日开始，Codex power users 也会在 2.5 天内烧完 weekly limits（aniketapanjwani），因此原生 token budgeting——pre-task estimation、per-branch attribution、automatic optimization——是最直接的机会。6 月 1 日后，每个 agent user 都可能成为客户。

**[+++] Agent Skills Marketplace and Discovery** -- Obsidian Skills（几天内 27K stars）、andrej-karpathy-skills（101K total，24K weekly）、Matt Pocock 的 engineering skills，以及 awesome-copilot repo，都显示对组合式 agent skills 的爆发需求。还没有统一市场用于跨 Claude Code、Codex、OpenCode 发现、评分和组合 skills。`npx skills add` 模式已经存在，但发现仍靠 Twitter 口碑。

**[++] Visual Workspace for Agentic Coding** -- [@nimbalyst](https://x.com/nimbalyst) 的 [开源发布](https://x.com/nimbalyst/status/2049849943133749541)为 Codex/Claude Code/OpenCode 提供 markdown、mockups、diagrams 和 diffs 的 visual editors，回应了终端智能体缺少视觉反馈的持续抱怨。Codex Desktop App UI 更好（aniketapanjwani）与 Claude Code 终端-only interface 之间，留下了一个可为任何智能体添加视觉层的工具缺口。

**[+] Provider-Agnostic Harness Benchmarking** -- [@caspar_br](https://x.com/caspar_br) 的 [deepagents profiles](https://x.com/caspar_br/status/2049672406260924729)通过按提供商调优 harness 带来 10-20 point 改进。大多数开发者仍使用默认配置。一个能在你的具体代码库上 benchmark 多个 harness profiles，并推荐最优设置的工具，可以捕获目前需要手动实验才能获得的价值。

## 8. 要点总结

1. **Microsoft 的 AI 经济学面临公开信誉挑战。** [@edzitron](https://x.com/edzitron) 的 [批评](https://x.com/edzitron/status/2049885224738472366)（204 点赞，11,543 浏览量）把叙事从“Copilot pricing changes”改写为“$300B 投资是否曾经可行”。转向 token-based billing 正被解读为 unit economics 崩坏的承认，而不是常规调价。这给了每个反 Microsoft lock-in 的替代品弹药。

2. **Codex 正从编程工具转向通用工作平台。** [role picker](https://x.com/aakashgupta/status/2049966530822332502)（Engineering 到 Student）、[CLI 0.128.0](https://x.com/fcoury/status/2049917871799636201) 中的 /goal 持久化，以及 GPT-5.5 launch party，都说明 OpenAI 正把 Codex 构建成横向平台。[@blakeandersonw](https://x.com/blakeandersonw) 的 [反驳](https://x.com/blakeandersonw/status/2049975392841158938)称 Claude 在“tier 2-3 knowledge workers”中动能强得多，说明竞争目标是非开发者多数。

3. **Agent skills 是新的价值创造层。** Obsidian Skills（27K stars）、andrej-karpathy-skills（101K stars，24K/week）、free-claude-code（85% weekly growth）和 Matt Pocock 的 engineering skills 表明，真正的差异化不再是你用哪个模型，而是你挂了哪些 skills。token optimization repos（RTK、Context Mode）说明 skills 正在解决 agents 自身没有解决的问题。

4. **Harness engineering 正在产品化。** [@caspar_br](https://x.com/caspar_br) 的 [deepagents profiles](https://x.com/caspar_br/status/2049672406260924729)（10-20 point gains）和 Cursor 的 per-model harness tuning（Quan2m2）把 harness optimization 从 hacker culture（4 月 29 日的 goblin mode）带进主流工具。这个 insight 已经成为共识：same model + better harness = meaningfully better output。

5. **Antigravity 的可靠性侵蚀进入第六天。** [@amritwt](https://x.com/amritwt)（161 点赞）、[@TheGlobalMinima](https://x.com/TheGlobalMinima)、[@y_qecea](https://x.com/y_qecea)（14,127 浏览量）和缺失 Opus 4.7 model（luicho9_）延续了此前模式。$20/月 value proposition（immasiddx，32 点赞）仍是唯一辩护，但前提是服务真的可用。

6. **Codex vs Claude Code 的对比现在是功能级，而不是模型级。** [@aniketapanjwani](https://x.com/aniketapanjwani) 的 [详细对比](https://x.com/aniketapanjwani/status/2049661842373451855)（29 收藏数）——subagent limits、permission modes、plugin architecture、environment bootstrapping——说明竞争已经从“哪个模型代码写得更好”转向“哪个 harness 更能处理复杂工作流”。下一波差异化会发生在这里。
